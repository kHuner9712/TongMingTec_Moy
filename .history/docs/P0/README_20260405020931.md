## 文档概览

本目录包含 MOY（墨言）项目的 P0 阶段完整文档体系，用于项目立项、需求确认和开发启动。

**项目定位：** 企业级 AI 原生客户经营系统

**核心链路：** 获客 → 沟通 → 谈单 → 成交 → 售后 → 复购

**首期策略：** Web 端优先，首期完整版包含 P0 和 P1 全部功能

## 首期完整版范围定义

### 范围术语说明

| 术语 | 定义 | 包含内容 |
|------|------|----------|
| **首期完整版** | 第一个可交付的生产版本 | P0全部功能 + P1全部功能 |
| **P0** | 核心业务功能 | 客户管理、线索管理、会话管理、商机管理、工单管理 |
| **P1** | 增强业务功能 | 知识库管理、数据看板 |
| **P2** | 后续迭代功能 | 高级分析、扩展集成等（不在首期范围） |

### P0 核心功能

| 模块 | 功能点 | 说明 |
|------|--------|------|
| 客户管理 | 客户档案、客户列表、客户详情、客户标签 | 核心数据实体 |
| 线索管理 | 线索录入、线索分配、线索跟进、线索转化 | 销售获客入口 |
| 会话管理 | 多渠道接入、会话列表、消息收发、智能回复 | 客服核心场景 |
| 商机管理 | 商机创建、阶段推进、商机预测、商机分析 | 销售核心场景 |
| 工单管理 | 工单创建、工单流转、SLA管理、工单统计 | 售后服务场景 |

### P1 增强功能

| 模块 | 功能点 | 说明 |
|------|--------|------|
| 知识库 | 知识分类、知识条目、知识检索、AI问答 | AI能力支撑 |
| 数据看板 | 销售看板、客服看板、运营看板 | 数据分析支撑 |

## 文档冲突解决规则

当文档间存在冲突时，按以下优先级解决：

| 冲突类型 | 主基线 | 跟随修订 | 解决规则 |
|----------|--------|----------|----------|
| 业务需求冲突 | BRD | PRD/RTM | BRD为业务源头 |
| 产品需求冲突 | PRD | HLD/API/DBD | PRD为产品定义权威 |
| 状态机定义冲突 | PRD 8.6章节 | DBD/API | PRD状态机为业务定义 |
| 数据模型冲突 | DBD | API/代码 | DBD为数据层权威 |
| 接口契约冲突 | API文档 | 前后端代码 | API为联调基线 |
| 技术架构冲突 | HLD | 代码实现 | HLD为架构权威 |
