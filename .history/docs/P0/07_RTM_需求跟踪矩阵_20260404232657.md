## 一、文档目的

本文档作为 MOY 项目的**需求跟踪底稿**，用于：

1. 建立业务需求 → 产品需求 → 开发任务 → 测试用例 → 验收点的完整追溯链
2. 确保每个需求可追踪、可验证、可交付
3. 支持需求变更影响分析
4. 作为开发、测试、验收的工作底稿

**使用规则：**
- 开发阶段：根据需求 ID 开发，完成后更新状态
- 测试阶段：根据测试点编写测试用例，完成后更新状态
- 验收阶段：根据验收点验收，完成后更新状态
- 需求变更：更新本矩阵，评估影响范围

---

## 二、跟踪矩阵说明

### 2.1 矩阵结构

```
业务需求（BRD）→ 产品需求（PRD）→ 页面/模块 → 开发任务 → 测试点 → 验收点
```

### 2.2 字段说明

| 字段 | 说明 |
|------|------|
| 需求ID | 唯一标识，格式：REQ-模块-序号 |
| 业务需求 | 来自 BRD 的业务需求描述 |
| 产品需求 | 来自 PRD 的产品需求描述 |
| 页面/模块 | 对应的页面或模块 |
| 开发任务 | 开发任务分解 |
| 测试点 | 测试验证点 |
| 验收点 | 上线验收标准 |
| 责任角色 | 负责的角色 |
| 优先级 | P0/P1/P2 |
| 状态 | 待开发/开发中/待测试/测试中/已验收 |
| 变更记录 | 需求变更历史 |
