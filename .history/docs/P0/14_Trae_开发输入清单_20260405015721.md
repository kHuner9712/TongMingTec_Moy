## 一、文档使用说明

### 1.1 使用目的

本文档为使用 Trae 进行代码生成的开发人员提供**最小必要输入**，确保生成的代码与产品定义一致。

### 1.2 使用原则

1. **先读文档，后生成代码**：每次生成代码前必须先阅读相关文档章节
2. **按模块使用**：根据当前开发的模块，阅读对应的章节
3. **禁止自行发挥**：不得生成文档未定义的功能、接口、字段

| 状态（中文） | 状态（英文枚举） | 允许流转目标       |
| ------------ | ---------------- | ------------------ |
| 新线索       | new              | assigned, invalid  |
| 已分配       | assigned         | following, invalid |

| 已转化       | converted        | 终态               |
| 已失效       | invalid          | 终态               |

**流转规则：**

- new → assigned：分配线索
- new → invalid：直接标记无效
- assigned → following：添加首次跟进
- assigned → invalid：标记无效
- following → converted：转化为商机（必须关联客户）
- following → invalid：标记无效

### 5.5 阶段流转

| 阶段（中文） | 阶段（英文枚举） | 默认赢率 | 允许流转目标      |
| ------------ | ---------------- | -------- | ----------------- |
| 初步接触     | initial          | 10%      | requirement, lost |
| 需求确认     | requirement      | 30%      | proposal, lost    |
| 方案报价     | proposal         | 50%      | negotiation, lost |
| 商务谈判     | negotiation      | 70%      | won, lost         |
| 成交         | won              | 100%     | 终态              |
| 失败         | lost             | 0%       | 终态              |

**流转规则：**

- 只能推进到下一阶段，不能跳跃
- 任何阶段都可以标记为失败
- 只有"商务谈判"阶段可以标记成交
