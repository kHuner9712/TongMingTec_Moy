## 文档目的

本文档面向产品、设计、开发、测试团队，从**产品实现视角**定义 MOY 首期 MVP 的详细需求，包括：

1. 产品概述与 MVP 定位
2. 用户故事与功能需求
3. 主流程与分支流程
4. 页面清单与字段说明
5. 业务规则与校验逻辑
6. 非功能需求
7. 验收标准

**阅读建议：**
- 产品团队：全文阅读
- 设计团队：重点阅读页面清单、字段说明
- 开发团队：重点阅读用户故事、业务规则、校验逻辑

**重要说明：** 本文档范围限定为首期 MVP，不包含后续规划功能。

## 二、产品概述

### 2.1 产品定位

| 维度 | 定义 |
|------|------|
| 产品名称 | MOY（墨言） |
| 产品定位 | 企业级 AI 原生客户经营系统 |
| 核心价值 | 不是"会聊天的 AI 工具"，而是"可交付经营结果的系统" |
| 目标用户 | 销售型公司、客服团队 |
| 终端形态 | Web 端（SaaS） |

### 2.2 MVP 核心目标

| 目标 | 说明 |
|------|------|
| 产品目标 | 完成客户经营核心链路的 MVP 版本，实现销售与客服一体化 |
| 客户目标 | [Assumption] 首年服务 50 家付费客户 |
| 业务目标 | 验证产品价值假设，建立可复制的销售与服务模式 |

## 三、目标用户

### 3.1 用户角色

| 角色 | 说明 | 使用场景 |
|------|------|----------|
| 管理员 | 企业管理员，管理系统配置 | 用户管理、权限分配、系统设置 |

| 角色 | 权限范围 |

| 管理员 | 全部权限 |

## 五、主流程与分支流程

### 5.1 线索转商机流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    线索转商机流程                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  录入线索 ──▶ 分配线索 ──▶ 跟进线索 ──▶ 转化为商机           │
│     │           │           │              │                    │
│     ▼           ▼           ▼              ▼                    │
│  填写信息   系统分配/    记录跟进    选择客户创建商机           │
│             手动分配     跟进记录    自动带出客户信息           │
│                                                                 │
│  【分支】                                                    │
│     │                                                        │
│     ├── 跟进中发现客户无意向 ──▶ 标记为"已失效"              │
│     │                                                        │
│     └── 跟进中客户主动咨询 ──▶ 转入"会话管理"处理            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 客户服务流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    客户服务流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  客户咨询 ──▶ 接入会话 ──▶ 处理问题 ──▶ 创建工单（如需）     │
│     │           │           │              │                    │
│     ▼           ▼           ▼              ▼                    │
│  多渠道接入   系统分配/    知识检索/    填写问题信息           │
│             手动分配     AI辅助/       选择处理人             │
│                          人工处理                           │
│                                                                 │
│  【分支】                                                    │
│     │                                                        │
│     ├── 问题已解决 ──▶ 关闭会话 ──▶ 邀请满意度评价           │
│     │                                                        │
│     └── 问题未解决 ──▶ 创建工单 ──▶ 进入工单处理流程         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 十、异常处理

### 10.1 会话异常

| 异常场景 | 处理方式 |
|----------|----------|
| 客服离线时新会话 | 将会话放入队列，等客服上线后自动分配 |
| 会话中断（网络问题） | 自动保存草稿，客服重新进入可继续 |
| 客户长时间未响应 | 超过 5 分钟未回复，提醒客服可结束会话 |

### 10.2 工单异常

| 异常场景 | 处理方式 |
|----------|----------|
| 处理人旷工/离职 | 工单自动释放，可重新分配 |
| 工单超时 | 自动升级至主管，并发送通知 |
| 客户重复提交 | 提示已有工单在处理中，可选择合并或新建 |

### 10.3 系统异常

| 异常场景 | 处理方式 |
|----------|----------|
| AI 服务不可用 | 降级为纯人工服务，显示提示 |
| 数据库连接失败 | 显示错误页面，提示稍后重试 |
| 文件上传失败 | 提示上传失败原因，支持重试 |
