本文档面向管理层和业务方，从**业务价值视角**定义 MOY 项目的需求，包括：

3. 核心业务场景与流程
4. 业务功能范围

**阅读建议：**

## 二、业务背景与产品定位

### 2.1 业务背景

#### 2.1.1 市场机会

| 维度 | 现状 | 机会 |
|------|------|------|
| 市场规模 | 2024年中国SaaS市场规模1026-1201亿元，持续增长 | 市场空间足够大 |
| AI 渗透 | 71% 组织已部署 GenAI，AI 正成为 SaaS 基础能力 | AI-Native 产品机会 |
| 竞争格局 | 国产 CRM 首次超越国际品牌，但多为单点工具 | 一体化解决方案空白 |

#### 2.1.2 客户痛点

| 痛点类别 | 具体表现 | 业务影响 |
|----------|----------|----------|
| 线索流失 | 售前线索留资率不足 15% | 大量意向客户直接流失 |
| 跟进无闭环 | 信息零散存储，无法统一管理 | 转化率低，依赖个人能力 |
| 知识沉淀难 | 新员工 60% 依赖"翻手册"或"问同事" | 培训成本高，效率低 |
| 服务不标准 | 22% 客户反馈"同样问题不同解决方案" | 客户信任度下降 |
| 工具碎片化 | 客服需反复切换多个系统 | 专注度与生产力下降 |
| 数据孤岛 | 销售、客服数据不互通 | 客户画像不完整 |

### 2.2 产品定位

#### 2.2.1 定位声明

**MOY（墨言）** 是企业级 AI 原生客户经营系统，围绕"获客-沟通-谈单-成交-售后-复购"全链路，以业务结果交付为核心价值主张。

**核心差异化：**

| 维度 | 传统方案 | MOY 方案 |
|------|----------|----------|
| 产品形态 | 单点工具（CRM 或客服系统） | 全链路系统 |
| AI 定位 | 聊天助手（附加功能） | 执行引擎（核心能力） |
| 价值交付 | 卖功能 | 交付业务结果 |
| 数据管理 | 数据孤岛 | 统一客户视图 |
| 销售-客服关系 | 割裂 | 一体化经营链路 |

#### 2.2.2 价值主张

**不是"会聊天的 AI 工具"，而是"可交付经营结果的系统"。**

| 价值维度 | 具体价值 | 量化目标（待验证） |
|----------|----------|---------------------|
| 提升转化 | 线索闭环管理，减少流失 | [Inference] 转化率提升 10-30% |
| 提升效率 | AI 辅助响应，知识沉淀 | [Inference] 客服效率提升 30%+ |
| 缩短周期 | 商机可视化，流程标准化 | [Inference] 成交周期缩短 20%+ |
| 提升满意度 | 服务标准化，响应及时 | [Inference] 满意度提升 15%+ |
| 提升复购 | 流失预警，客户经营 | [Inference] 复购率提升 10%+ |

| 高级报表 | 首期聚焦核心指标 | P2 |
| 营销自动化 | 首期聚焦核心链路 | P2 |
| 视频会议 | 非首期核心需求 | P3 |

### 5.2 功能模块详细说明

#### 5.2.1 客户管理

| 维度 | 说明 |
|------|------|
| 业务目标 | 建立统一客户视图，消除数据孤岛 |
| 目标用户 | 销售人员、客服人员、管理者 |
| 核心功能 | 客户档案、客户画像、客户分组、客户标签 |
| 业务价值 | 客户信息统一管理，支撑精准营销和服务 |

**业务规则：**
- 客户档案包含基本信息、联系方式、企业信息
- 客户画像整合销售、服务、行为数据
- 客户分组支持按行业、规模、阶段等维度
- 客户标签支持自定义，便于精准触达

#### 5.2.2 线索管理

| 维度 | 说明 |
|------|------|
| 业务目标 | 线索闭环管理，提升转化率 |
| 目标用户 | 销售人员、销售管理者 |
| 核心功能 | 线索录入、线索分配、线索跟进、线索转化 |
| 业务价值 | 线索不流失，跟进及时，转化可追踪 |

**业务规则：**
- 线索来源：手动录入、表单导入、渠道接入
- 分配规则：支持自动分配（轮询、负载均衡）和手动分配
- 跟进要求：新线索 24 小时内首次跟进
- 转化标准：线索转化为商机或标记为无效

#### 5.2.3 会话管理

| 维度 | 说明 |
|------|------|
| 业务目标 | 多渠道统一接入，提升响应效率 |
| 目标用户 | 客服人员、销售人员 |
| 核心功能 | 多渠道接入、会话记录、智能回复、话术辅助 |
| 业务价值 | 响应及时，服务标准，沟通高效 |

**业务规则：**
- 支持渠道：Web 在线客服、微信、企业微信（首期）
- 会话分配：支持自动分配和手动分配
- 智能回复：AI 辅助回复常见问题
- 话术辅助：根据场景推荐话术模板

#### 5.2.4 商机管理

| 维度 | 说明 |
|------|------|
| 业务目标 | 商机可视化，缩短成交周期 |
| 目标用户 | 销售人员、销售管理者 |
| 核心功能 | 商机创建、商机阶段、商机跟进、商机预测 |
| 业务价值 | 商机状态透明，推进标准化，预测准确 |

**业务规则：**
- 商机阶段：初步接触 → 需求确认 → 方案报价 → 商务谈判 → 成交/失败
- 跟进要求：每个商机每周至少 1 次跟进
- 预测依据：基于商机阶段和历史转化率

#### 5.2.5 工单管理

| 维度 | 说明 |
|------|------|
| 业务目标 | 服务标准化，提升客户满意度 |
| 目标用户 | 客服人员、客服管理者 |
| 核心功能 | 工单创建、工单流转、工单统计、工单评价 |
| 业务价值 | 问题追踪闭环，服务可量化，协作高效 |

**业务规则：**
- 工单类型：咨询、投诉、售后、其他
- 优先级：紧急、高、中、低
- SLA：紧急 2 小时响应，高 4 小时，中 8 小时，低 24 小时
- 流转规则：支持跨部门流转，责任人明确

#### 5.2.6 知识库

| 维度 | 说明 |
|------|------|
| 业务目标 | 知识沉淀，降低培训成本 |
| 目标用户 | 客服人员、销售人员、管理者 |
| 核心功能 | 知识录入、知识分类、知识检索、AI 问答 |
| 业务价值 | 知识快速查找，培训效率提升 |

**业务规则：**
- 知识类型：FAQ、产品文档、流程规范、话术模板
- 知识审核：新知识需审核后发布
- AI 问答：基于知识库智能回答问题

#### 5.2.7 数据看板

| 维度 | 说明 |
|------|------|
| 业务目标 | 经营可视化，支撑决策 |
| 目标用户 | 管理者 |
| 核心功能 | 核心指标展示、趋势分析、对比分析 |
| 业务价值 | 数据驱动决策，问题及时发现 |

**业务规则：**
- 核心指标：线索转化率、首响时效、跟进及时率、成交周期、客户满意度、复购率
- 更新频率：实时或每日更新
- 权限控制：按角色查看不同维度数据

## 七、业务指标与验收标准

### 7.1 北极星指标

| 指标 | 定义 | 计算公式 | 目标值（待验证） |
|------|------|----------|------------------|
| 线索转化率 | 成交客户数 / 线索总数 | 成交客户数 ÷ 线索总数 × 100% | [Inference] 提升 10-30% |
| 首响时效 | 首次响应时间 | 从客户咨询到首次响应的时间 | [Inference] 缩短 50%+ |
| 跟进及时率 | 及时跟进次数 / 应跟进次数 | 及时跟进次数 ÷ 应跟进次数 × 100% | > 90% |
| 成交周期 | 从首次接触到成交的平均天数 | Σ 成交天数 ÷ 成交客户数 | [Inference] 缩短 20%+ |
| 客户满意度 | CSAT 评分 | 满意评价数 ÷ 总评价数 × 100% | > 85% |
| 复购率 | 续费客户数 / 应续费客户数 | 续费客户数 ÷ 应续费客户数 × 100% | [Inference] 提升 10%+ |

### 7.2 过程指标

| 指标 | 定义 | 监控频率 | 目标值 |
|------|------|----------|--------|
| 线索录入量 | 新增线索数量 | 每日 | 按客户规模设定 |
| 线索跟进率 | 已跟进线索 / 总线索 | 每日 | > 95% |
| 商机创建量 | 新增商机数量 | 每日 | 按客户规模设定 |
| 商机推进率 | 推进到下一阶段的商机比例 | 每周 | > 30% |
| 工单解决率 | 已解决工单 / 总工单 | 每日 | > 95% |
| 工单及时解决率 | 在 SLA 内解决的工单比例 | 每日 | > 90% |
| 知识库使用率 | 使用知识库的会话比例 | 每日 | > 50% |

### 7.3 验收标准

#### 7.3.1 功能验收标准

| 功能模块 | 验收标准 | 验收方式 |

| 客户管理 | 客户档案完整录入、查询、分组功能正常 | 功能测试 |
| 线索管理 | 线索录入、分配、跟进、转化流程完整 | 功能测试 |

| 商机管理 | 商机创建、阶段管理、跟进功能正常 | 功能测试 |
| 工单管理 | 工单创建、流转、统计功能完整 | 功能测试 |
| 知识库 | 知识录入、检索、AI 问答可用 | 功能测试 |
| 数据看板 | 核心指标展示正常、数据准确 | 功能测试 |

#### 7.3.2 业务验收标准
