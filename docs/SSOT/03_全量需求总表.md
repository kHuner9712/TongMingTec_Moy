# MOY 全量需求总表

## 1. 文档定位
本文档是 MOY 唯一 REQ 入口。所有需求必须在本文档完成统一编码、阶段标注和影响链登记。

## 2. 编码归一与旧口径吸收
| 旧口径 | 新口径 | 处理方式 |
| --- | --- | --- |
| `REQ-SM-*` | `REQ-CNV-*` | 会话模块统一改归 `CNV` |
| `REQ-TM-*` | `REQ-TK-*` | 工单模块统一改归 `TK` |
| `REQ-USER-*` | `REQ-USR-*` | 用户/角色/权限统一改归 `USR` |
| `REQ-DB-*` | `REQ-DASH-*` | 业务看板统一改归 `DASH` |
| `REQ-MA-*` | `REQ-AUTO-*` | 营销自动化统一改归 `AUTO` |
| `REQ-OP-*` | `REQ-INT-*` | 开放平台统一改归 `INT` |
| `REQ-MB-*` | `REQ-PLT-*` | 移动端/多端统一改归 `PLT` |
| `REQ-ONPREM-*` | `REQ-DEPLOY-*` | 私有化/部署统一改归 `DEPLOY` |
| `REQ-AIQC-*` | `REQ-AI-*` | AI 质检统一纳入 `AI` |
| `REQ-CS-*` | `REQ-CSM-*` | 客户成功统一改归 `CSM` |

## 3. 字段说明
| 字段 | 说明 |
| --- | --- |
| `introduced_in` | 首次进入 SSOT 的阶段 |
| `required_in` | 到该阶段必须具备 |
| `must_for_release` | 是否为该阶段发布门禁 |
| `release_scope` | 需要参与的阶段集合 |
| `status` | `validated/specified/planned` 之一；S1 继承 P0 为 `validated` |
