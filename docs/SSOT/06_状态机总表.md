# MOY App 状态机总表

## 1. 文档定位
本文档是 MOY App 唯一状态机基线，定义：
- 状态枚举
- 初始状态
- 终止状态
- 合法迁移 / 非法迁移
- 触发 API / Action
- 权限要求
- 审计要求
- 阶段适用范围

## 2. 统一规则
- 状态机 ID 统一为 `SM-{entity_snake_case}`。
- 所有状态迁移必须经过服务端校验，前端仅做按钮显隐。
- 非法迁移统一返回 `STATUS_TRANSITION_INVALID`。
- 涉及核心写操作的迁移必须写入 `TABLE-AUD-001`。
- S1 已实现状态机继承 P0 工程口径；S2+ 在不破坏 S1 约束的前提下扩展。
