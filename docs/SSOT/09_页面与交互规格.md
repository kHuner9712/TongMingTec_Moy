# MOY App 页面与交互规格

## 1. 文档定位
本文档是 MOY App 唯一 PAGE 主文档，统一定义：
- PAGE 编码
- 路由
- 终端归属
- 页面目标与区块
- 表格列 / 筛选 / 表单 / 按钮
- 权限显隐 / 状态显隐
- 调用 API
- 空态 / 错态 / 加载态 / 无权限态
- WebSocket / 实时刷新点

### 1.1 实现级展开入口
- `09` 主文继续承担 PAGE 清单、全局交互规则与页面族模板的唯一主链职责。
- 终局关键页面的逐页交互合同，统一以下列实现级子文档为准：
  - [09A_关键页面逐页交互展开.md](./09A_关键页面逐页交互展开.md)
- `09A` 已把会话、工单、商机、报价、合同、订单、订阅、账单、支付、发票、AI、开放平台、多端、国际化、部署等关键页面补到可直接生成 React 页面的粒度。

## 2. 全局交互规则
- 首页 `/` 只允许重定向到 `/dashboard`
- 路由参数统一 `{id}`
- 所有页面必须具备四态：
  - `loading`: `Skeleton + Spin`
  - `empty`: `Empty`
  - `error`: `Result(status='error')`
  - `forbidden`: `Result(status='403')`
- 所有写操作成功后：
  - 弹 `message.success`
  - 精准刷新当前页或当前区块
- 所有终态动作必须二次确认：
  - 关闭会话
  - 关闭工单
  - 赢/输单
  - 提交审批
  - 发票开具
  - 停服 / 恢复

## 3. 页面族模板
### 3.1 列表页模板
- 区块：筛选区 / 表格区 / 分页区 / 主操作
- 通用列：编号、状态、负责人、更新时间、操作
- 通用筛选：`keyword`, `status`, `owner`, `created_at`
- 成功反馈：创建或更新后刷新当前列表

### 3.2 详情页模板
- 区块：摘要卡 / 基本信息 / 时间线 / 子资源 / 右侧动作栏
- 通用操作：编辑、状态推进、审批、导出
- 详情页必须直接显示 `version` 对应的最新状态

### 3.3 工作台/流程页模板
- 区块：筛选区 / 看板区 / 右侧详情抽屉 / 实时事件流
- 适用：会话监控、AI 工作台、续费工作台、自动化执行页

## 5. 页面显隐规则
- 页面级显隐：必须先过必要权限判断。
- 按钮级显隐：必须同时满足权限 + 状态机。
- Portal / 客户端页面：同时受 `terminal_profile.feature_flags` 控制。
- 商业化后台页面：默认只对 `ROLE-ADMIN`、`ROLE-FINANCE_ADMIN`、`ROLE-CSM_MANAGER` 开放相应子页。

## 6. 实时刷新规则
- 会话、工单、通知、账单、订阅、AI 任务是首批实时对象。
- 实时事件到达后只刷新受影响区块，不全页重载。
- WebSocket 消费失败时，页面退化为手动刷新，不阻断主要操作。

## 10. AUTO 页面补充（2026-04）

### 10.1 页面目标
- `PAGE-AUTO-004`：`/automation`
- 页面定位：从“规则配置器”升级为“结果链动作闭环工作台”，支持推荐模板安装、运行追踪、失败定位、业务对象回溯与人工接管/确认。

### 10.2 页面区块
- 推荐模板区：展示模板说明、触发事件、步骤清单、安装/更新状态。
- 运行记录区：展示运行状态、触发来源、失败原因、审批态、业务对象链接。
- 步骤明细抽屉：展示每一步的审批请求、执行输出、失败原因、业务对象链接。
- 触发器区：保留通用触发器管理能力。
- 流程区：保留流程创建/编辑/手动执行能力。

### 10.3 关键交互
- 安装模板：点击后调用模板安装 API，自动生成/更新对应 event flow 并默认启用。
- 运行详情：可查看触发条件快照、步骤输出、审批请求 ID、失败原因。
- 人工接管：对 `running/awaiting_approval` 运行可执行接管，写入接管人和接管时间。
- 人工确认：对已结束运行可执行确认，写入确认人和确认备注。
- 业务跳转：从运行或步骤输出直接跳转合同/订单/订阅/交付/客户成功视图。

### 10.4 API 映射
- 推荐模板：`GET /automation/templates`，`POST /automation/templates/{code}/install`
- 运行记录：`GET /automation/runs`，`GET /automation/runs/{runId}/steps`
- 人工边界：`POST /automation/runs/{runId}/takeover`，`POST /automation/runs/{runId}/confirm`
- 兼容保留：`GET /automation/triggers`，`GET /automation/flows`，`POST /automation/flows/{id}/execute`

### 10.5 状态与反馈
- 运行状态：`pending/running/awaiting_approval/completed/failed/cancelled`
- 步骤状态：`pending/running/awaiting_approval/completed/failed/skipped`
- 失败反馈：显示 `error_code + error_message`，并支持展开查看步骤级失败细节。
- 审批反馈：高风险步骤进入 `awaiting_approval`，审批通过后自动恢复执行，拒绝后运行失败并保留证据。
