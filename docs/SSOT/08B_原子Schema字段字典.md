# MOY 原子 Schema 字段字典

## 1. 文档定位
本文档是 [08_API契约与Schema字典.md](./08_API契约与Schema字典.md) 的实现级子文档，与 [08A_原子API与Schema展开.md](./08A_原子API与Schema展开.md) 共同构成契约层最终实现输入。

约束：
- `08A` 负责原子 API 合同；本文负责模块级 schema 字段定义。
- 所有 `SCH-*` 都继承 `introduced_in / required_in / release_scope / status`。
- 为避免重复，`SCH-HEADER-*`、`SCH-PATH-ID`、`SCH-PAGE-QUERY`、`SCH-LIST-META`、`SCH-ACK`、`SCH-ERROR` 继续以 `08A` 为准。

记法：
- `字段:类型[长度]`；`!`=必填；`?`=可空；`=默认值`；`enum(...)`=枚举；`map(table.field)`=数据库字段映射。
- 同一行可定义多个 schema；这只是排版压缩，不是资源族占位。每个 schema 名称都必须视为已原子落地。
- `map=` 若指向运行态投影、provider session store、报表缓存、导出/导入作业等派生读写载体，仅表示实现来源，不构成新的 `TABLE-ID` 定义；凡需要进入 SSOT 对象清单的实体，仍必须以 `05/05A` 为准。
