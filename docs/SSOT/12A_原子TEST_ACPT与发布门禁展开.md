## 3. 原子 ACPT 矩阵
| ACPT-ID | business scenario / role | preconditions | steps | expected result | blocking rule / release gate |
| --- | --- | --- | --- | --- | --- |
| `ACPT-AUTH-001` | `登录与会话治理 / admin` | `S1 可用租户与 admin 用户已初始化` | `登录->查看当前身份->刷新 token->撤销旧会话` | `登录后进入 dashboard, 被撤销会话立即失效` | `任一鉴权接口 401/403 逻辑异常即阻塞 S1 发布; gate=AUTH smoke 必过` |
| `ACPT-CM-001` | `客户录入到激活 / 销售` | `有 PERM-CM-CREATE/UPDATE/STATUS` | `新建客户->补齐联系人->激活->列表检索` | `客户详情、列表、状态标签一致` | `客户状态与详情不一致即阻塞 CRM 发布; gate=CM CRUD + status` |
| `ACPT-LM-001` | `Lead -> Opportunity / 销售` | `线索 owner 已就位` | `创建 lead->分配->跟进->转化` | `生成 customer+opportunity, lead=converted` | `转化链任一 ID 缺失即阻塞 S1 发布; gate=Lead 主链必过` |
| `ACPT-CNV-001` | `Conversation -> Ticket / 客服` | `有 queued 会话` | `接入->回复->转工单->关闭会话->处理工单` | `会话与工单互有关联, WS 正常刷新` | `转工单后无 ticket.created 事件即阻塞服务域发布; gate=CNV/TK E2E` |
| `ACPT-TK-001` | `工单全状态闭环 / 客服班长` | `已有待处理工单与 SLA 配置` | `分派->开始->解决->关闭->查看 SLA 指标` | `工单状态、日志、SLA 指标一致` | `状态链断裂或 SLA 指标缺失即阻塞 TK 发布; gate=TK full flow` |
| `ACPT-OM-001` | `商机推进到赢单 / 销售经理` | `已由线索转出商机` | `推进阶段->查看预测->标记 won` | `阶段历史完整, result 只会是 won` | `出现非 won/lost 结果值即阻塞全仓发布; gate=OM result constraint` |
| `ACPT-AI-001` | `AI Suggest -> Human Confirm / 客服主管` | `AI provider 可用且审批人在线` | `请求 smart reply->生成审批->人工 approve->发送` | `AI 只在审批后执行, 审计链完整` | `高风险 AI 无审批即可落地则阻塞 S2+ 发布; gate=AI approval` |
| `ACPT-QT-001` | `Quote -> Contract -> Order / 商务` | `商机已到 negotiation; 审批流可用` | `创建报价->审批通过->发送->生成合同->签署->生成订单` | `quote/contract/order 编号与关联链完整` | `任一对象脱链即阻塞商业化发布; gate=QT/CT/ORD 主链` |
| `ACPT-CT-001` | `合同审批与签署 / 法务` | `已有 approved quote 与电子签 provider` | `创建合同->提交审批->审批通过->签署->做 expire-check` | `合同进入 active 或到期提醒可触发` | `审批/签署任一不可追踪即阻塞 CT 发布; gate=CT approve/sign` |
| `ACPT-ORD-001` | `订单激活 / 商业化运营` | `已有 active contract` | `创建订单->激活->检查订阅卡` | `订单激活后订阅已生成且状态正确` | `订单激活不产出订阅即阻塞 ORD 发布; gate=ORD activation` |
| `ACPT-SUB-001` | `Order -> Subscription -> Bill -> Payment / 财务运营` | `已存在可激活订单与计费配置` | `激活订单->生成订阅->生成账单->创建支付->对账` | `订阅、账单、支付状态联动正确` | `账单金额与支付金额不闭环即阻塞 S3 发布; gate=SUB/BILL/PAY 主链` |
| `ACPT-BILL-001` | `账单生成与催缴 / 财务运营` | `存在 active subscription 与 overdue 样本` | `生成账单->导出->催缴->查看停服建议` | `账单金额、导出文件、催缴记录一致` | `导出或催缴缺任一即阻塞 BILL 发布; gate=BILL collection` |
| `ACPT-PAY-001` | `支付创建与对账 / 财务` | `存在 open bill 与 provider ref` | `创建 payment->对账->检查 bill paid 联动` | `payment/bill 状态一致, 差异明细可追` | `对账后账单未闭合即阻塞 PAY 发布; gate=PAY reconcile` |

## 4. 发布门禁与迁移检查
| 主题 | 可执行检查项 | 阻塞条件 |
| --- | --- | --- |
| `阶段发布门禁` | `S1=AUTH/CM/LM/OM/CNV/TK 主链 TEST+ACPT 全绿; S2=AI/QT/CT/CSM/AUTO 补齐; S3=ORD/SUB/BILL/PAY/INV/INT/PLT 补齐; S4=I18N/DEPLOY/字段权限/onprem 补齐` | `任一 required_in<=当前阶段 的 ACPT 未通过即阻塞发布` |
| `migration 演练` | `核对 deployment_profile=ready; 备份快照完成; dry_run batch 成功; rollback_plan 已评审; migration_batches 有 batch_no/rollback_token` | `dry_run 失败、快照缺失、rollback_plan 缺失、批次无审计记录` |
| `rollback 前置` | `确认无不可逆 DDL; 财务表仅补偿不硬删; 对外 provider 回调已暂停或可幂等重放; license/token 不会重复绑定` | `存在不可逆 DDL 未备份、支付/发票已外发且无补偿方案` |
| `onprem 额外检查` | `license offline/online 模式匹配; secret_ref 已在目标机房下发; region/data residency 合规; 本地对象存储与备份演练通过` | `license 未激活、secret 缺失、数据驻留不满足、备份恢复未验证` |
| `i18n/region/compliance` | `locale_resources 发布覆盖率达标; region_policies 已绑定计费/税则; consent 记录链路可审计; 门户/邮件模板按 locale 渲染正确` | `缺失 key、区域税则未绑定、隐私同意不可追溯、跨区模板错语种` |
