# MOY 业务域与模块树

## 1. 文档目标
本文档定义 MOY 终局版的唯一业务域划分、模块树、模块编码与模块依赖。

## 2. 顶层业务域
| 业务域 | 说明 | introduced_in | required_in |
| --- | --- | --- | --- |
| 获客域 | 客户、线索、渠道、营销、分群、评分、回收 | S1 | S4 |
| 沟通域 | 会话、消息、接待、转接、评价、客户触点 | S1 | S4 |
| 商机域 | 商机、报价、合同、订单、支付、发票 | S1 | S4 |
| 服务域 | 工单、任务、通知、知识、SLA、客户门户 | S1 | S4 |
| 客户成功域 | 健康度、成功计划、续费、增购、预警 | S2 | S4 |
| AI Agent 域 | 智能回复、AI task、Agent registry、审批、回滚、接管 | S1 | S4 |
| 商业化域 | plan、addon、quota、subscription、bill、payment、invoice | S3 | S4 |
| 平台与开放生态域 | 集成、Webhook、开放 API、SDK、开发者中心 | S3 | S4 |
| 国际化/合规/私有化域 | 多语言、多区域、隐私、私有化、迁移 | S3 | S4 |

## 5. 模块依赖
| 模块 | 直接依赖 | 说明 |
| --- | --- | --- |
| AUTH | ORG, USR | 认证必须绑定租户与用户 |
| ORG | - | 组织是租户主键边界 |
| USR | ORG | 用户、角色、权限均以 `org_id` 隔离 |
| CM | ORG, USR | 客户归属租户，负责人是用户 |
| LM | ORG, USR, CM | 线索可转客户/商机 |
| CHN | ORG, USR, CNV | 渠道接入后汇入会话 |
| AUTO | CM, LM, CHN, NTF | 自动化触达基于客户/线索/渠道执行 |
| CNV | ORG, USR, CHN, CM | 会话绑定客户与渠道 |
| OM | CM, LM, USR | 商机源于客户与线索 |
| QT | OM, CM, USR | 报价依附商机 |
| CT | OM, QT, CM, USR | 合同通常由商机/报价生成 |
| ORD | CT, QT, CM | 订单承接合同/报价 |
| TK | CNV, CM, USR | 会话可转工单 |
| TSK | USR, TK, OM, CSM | 任务服务于多个业务模块 |
| NTF | USR | 通知投递给用户或客户端主体 |
| KB | ORG, USR | 知识按租户隔离 |
| DASH | CM, LM, OM, TK, CSM, BILL | 看板聚合业务数据 |
| CSM | CM, OM, CT, SUB, BILL | 客户成功依赖成交后经营数据 |
| AI | CNV, KB, LM, OM, TK, USR, AUD | AI 读取上下文并受权限/审计约束 |
| COR | CM, EventsModule | 客户经营记录聚合客户数据与领域事件 |
| CMEM | CM, COR | 客户记忆依赖客户数据与经营记录 |
| ART | CNV, CM, EventsModule | Agent 运行时依赖会话上下文与事件 |
| APC | ART | 审批中心依赖 Agent 运行时的审批引擎 |
| TKC | ART | 接管中心依赖 Agent 运行时的接管引擎 |
| RBC | ART | 回滚中心依赖 Agent 运行时的回滚引擎 |
| AUD | - | 接收全系统关键写操作审计 |
| SYS | ORG, USR | 系统配置按租户/环境生效 |
| PLAN | ORG | 套餐目录与额度策略按租户生效 |
| SUB | PLAN, ORD, ORG | 订阅从订单开通 |
| BILL | SUB, PLAN, PAY | 账单源于订阅与用量 |

| INT | AUTH, ORG, USR, CM, LM, OM, TK, CNV | 外部集成对接核心资源 |
| PLT | AUTH, INT, CNV, TK, CSM | 多端和平台控制台依赖核心服务 |
| I18N | ORG, SYS, PLT | 国际化配置影响终端和规则解释 |
| DEPLOY | ORG, SYS, PLAN, SUB | 部署形态影响租户开通、许可与迁移 |

## 6. 模块与阶段关系
| 模块 | release_scope | status | 说明 |
| --- | --- | --- | --- |
| AUTH~SYS + COR/CMEM/ART/APC/TKC/RBC（S1 基线） | `S1,S2,S3,S4` | implemented | AI 原生架构已实现 |
| KB, DASH, AUTO, QT, CT, CSM | `S2,S3,S4` | specified | 进入 SSOT，等待分阶段实现 |
| ORD, PLAN, SUB, BILL, PAY, INV, INT, PLT | `S3,S4` | specified | 进入终局业务闭环和平台化范围 |
| I18N, DEPLOY | `S4` | specified | 终局规模版主范围 |

## 7. 模块设计约束
- 所有业务表默认继承 `org_id` 多租户隔离。
- 所有写接口必须可追溯到对应 `REQ / API / PAGE / TEST`。
- 模块之间的依赖只能通过对象、状态机、API、事件四类契约表达，不允许另立文档体系。
- 商业化与平台化模块不得创建第二套独立命名体系，必须沿用本表模块码。
