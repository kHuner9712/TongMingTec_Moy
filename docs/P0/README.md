# MOY 项目 P0 文档体系

---

## 文档概览

本目录包含 MOY（墨言）项目的 P0 阶段完整文档体系，用于项目立项、需求确认和开发启动。

**项目定位：** 企业级 AI 原生客户经营系统

**核心链路：** 获客 → 沟通 → 谈单 → 成交 → 售后 → 复购

**首期策略：** Web 端优先，MVP 范围收敛

## 核心结论摘要

### 市场机会

| 维度 | 结论 | 来源 |
|------|------|------|
| 市场规模 | 2024年中国SaaS市场1026-1201亿元，持续增长 | [Fact] IDC |
| AI 渗透 | 71% 组织已部署 GenAI，AI 成为 SaaS 基础能力 | [Fact] 行业报告 |
| 市场空白 | 销售-客服一体化解决方案存在空白 | [Inference] 竞品分析 |

### 产品定位

| 维度 | 定义 |
|------|------|
| 核心定位 | 企业级 AI 原生客户经营系统 |
| 差异化 | 不是"会聊天的 AI 工具"，而是"可交付经营结果的系统" |
| 目标客户 | 销售型公司、客服团队 |

### MVP 范围

| 优先级 | 功能模块 |
|--------|----------|
| P0 | 客户管理、线索管理、会话管理、商机管理、工单管理 |
| P1 | 知识库、数据看板 |

### 立项建议

**建议：有条件立项**

**前置条件：**
1. 完成目标客户深度访谈，验证需求假设
2. 确认首期 MVP 功能边界与预算
3. 咨询法律顾问，确认 AIGC 合规要求
4. 组建核心团队（产品负责人、技术负责人）
