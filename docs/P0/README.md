## 3. 当前正式开发冻结范围

### 3.1 当前正式开发范围（P0，已确认）
- 认证与会话：`AUTH`
- 组织与用户权限：`ORG`、`USR`
- 客户管理：`CM`
- 线索管理：`LM`
- 商机管理：`OM`
- 会话中心：`CNV`
- 工单管理：`TK`
- 任务管理：`TSK`
- 通知中心：`NTF`
- 渠道管理：`CHN`
- AI 智能回复任务：`AI`（仅 P0 轻量能力）
- 审计与系统能力：`AUD`、`SYS`（含仪表盘汇总与系统配置）

### 3.2 不进入当前正式开发范围（P1，暂不纳入）
- `KB` 知识库管理
- `DASH` 数据看板
- `AUTO` 自动化规则
- `AIOPS` AI 工作台高级能力

### 3.3 保留规划但不实现范围（P2）
- 合同、订单、发票、续费
- 呼叫中心与语音渠道
- 移动端（App/小程序）

## 5. 统一编号规范（全仓唯一）
| 类型 | 规则 | 示例 |
| --- | --- | --- |
| 需求 | `REQ-{MOD}-{NNN}` | `REQ-CM-001` |
| 页面 | `PAGE-{MOD}-{NNN}` | `PAGE-CNV-002` |
| 接口 | `API-{MOD}-{NNN}` | `API-TK-006` |
| 权限 | `PERM-{MOD}-{ACTION}` | `PERM-CNV-TRANSFER` |
| 测试 | `TEST-{MOD}-{NNN}` | `TEST-OM-003` |
| 验收 | `ACPT-{MOD}-{NNN}` | `ACPT-LM-004` |
| 开发任务 | `TASK-{MOD}-{NNN}` | `TASK-CNV-002` |
| 数据表 | `TABLE-{MOD}-{NNN}` | `TABLE-CM-001` |
| 状态机 | `SM-{ENTITY}` | `SM-opportunity` |

补充规则：
- 路径参数一律使用 `{id}`，禁止 `:id`。
- 接口 ID 与 HTTP 方法解耦，方法以 API 文档“Method 字段”为准。
- 根路由 `/` 只允许重定向到 `/dashboard`，`/dashboard` 为唯一首页路由。
