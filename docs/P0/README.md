## 文档概览

本目录包含 MOY（墨言）项目的 P0 阶段完整文档体系，用于项目立项、需求确认和开发启动。

**项目定位：** 企业级 AI 原生客户经营系统

**核心链路：** 获客 → 沟通 → 谈单 → 成交 → 售后 → 复购

**首期策略：** Web 端优先，当前正式开发冻结范围为 P0 全部功能，P1 暂不进入当前正式开发

## 正式开发冻结范围定义

### 范围术语说明

| 术语                   | 定义                             | 包含内容                                                       |
| ---------------------- | -------------------------------- | -------------------------------------------------------------- |
| **当前正式开发冻结范围** | 可立即启动开发的范围             | P0全部功能                                                     |
| **P0**                 | 核心业务功能（当前正式开发）     | 客户管理、线索管理、会话管理、商机管理、工单管理、任务管理、通知管理、渠道管理 |

## 文档冲突解决规则

当文档间存在冲突时，按以下优先级解决：

| 冲突类型       | 主基线      | 跟随修订    | 解决规则            |
| -------------- | ----------- | ----------- | ------------------- |
| 业务需求冲突   | BRD         | PRD/RTM     | BRD为业务源头       |
| 产品需求冲突   | PRD         | HLD/API/DBD | PRD为产品定义权威   |
| 状态机定义冲突 | PRD 8.6章节 | DBD/API     | PRD状态机为业务定义 |
| 数据模型冲突   | DBD         | API/代码    | DBD为数据层权威     |
| 接口契约冲突   | API文档     | 前后端代码  | API为联调基线       |
| 技术架构冲突   | HLD         | 代码实现    | HLD为架构权威       |
