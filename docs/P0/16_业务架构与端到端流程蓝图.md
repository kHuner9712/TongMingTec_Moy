## 一、文档目的

本文档作为 MOY 项目的**业务架构基线**，用于：

1. 明确目标客户细分与业务模式差异
2. 定义完整的客户经营链路
3. 统一销售与客服为一体化经营体系
4. 定义关键业务对象及其关系
5. 补充例外流程与异常处理
6. 提供端到端业务流程示例

**核心原则：** 销售与客服不是两个割裂系统，而是同一条客户经营链上的不同阶段能力。

## 二、目标客户细分

### 2.1 目标行业定义

#### 2.1.1 首批重点支持行业

| 行业 | 行业特征 | 典型业务模式 | 优先级 | 支持理由 |
|------|----------|--------------|--------|----------|
| **教育培训** | 销售驱动、咨询密集、复购周期明确 | 电销+顾问式销售、售后答疑 | P0 | 销售流程标准化程度高，客服需求明确 |
| **企业服务** | B2B销售、决策链长、客单价高 | 顾问式销售、客户成功导向 | P0 | 客户经营需求强烈，价值感知明显 |
| **软件/SaaS** | 订阅模式、续费关键、技术支持需求 | 网销+电销、技术客服 | P0 | 与MOY产品形态契合，易产生共鸣 |
| **金融服务** | 合规要求高、风险敏感、服务标准化 | 电销+渠道、合规客服 | P1 | 客单价高，但合规要求复杂 |
| **医疗健康** | 专业性强、隐私要求高、信任敏感 | 顾问式销售、专业客服 | P1 | 需求明确，但行业特殊性需适配 |

#### 2.1.2 暂不优先支持行业

| 行业 | 暂不支持原因 | 计划纳入阶段 |
|------|--------------|--------------|
| 零售电商 | 订单量大、物流协同复杂、需要ERP集成 | P2 |
| 制造业 | 供应链协同、生产排程、复杂BOM | P2 |
| 房地产 | 销售周期极长、渠道分销复杂 | P3 |
| 政务/公共事业 | 采购流程特殊、合规要求极高 | P3 |

### 2.2 目标企业规模定义

#### 2.2.1 企业规模分层

| 规模层级 | 员工人数 | 年营收 | 销售团队规模 | 客服团队规模 | 优先级 |
|----------|----------|--------|--------------|--------------|--------|
| **小微企业** | 10-50人 | <1000万 | 3-10人 | 1-5人 | P1 |
| **中小企业** | 50-200人 | 1000万-5000万 | 10-30人 | 5-15人 | P0 |
| **中型企业** | 200-500人 | 5000万-2亿 | 30-100人 | 15-50人 | P0 |
| **大型企业** | >500人 | >2亿 | >100人 | >50人 | P2 |

#### 2.2.2 首批目标客户画像

| 维度 | 定义 |
|------|------|
| 核心目标 | 中小企业（50-200人），销售团队10-30人，客服团队5-15人 |
| 次要目标 | 中型企业（200-500人），已有CRM或客服系统但效果不佳 |

### 2.3 典型组织形态

#### 2.3.1 销售型组织形态

| 组织形态 | 特征 | 适用场景 | 系统支持重点 |
|----------|------|----------|--------------|
| **电销团队** | 电话为主、高频触达、标准化话术 | 教育培训、金融服务 | 线索批量导入、自动分配、通话记录 |
| **网销团队** | 在线咨询、即时响应、多渠道触达 | 软件SaaS、电商服务 | 会话管理、智能回复、渠道接入 |
| **顾问式销售** | 深度沟通、方案定制、长周期跟进 | 企业服务、咨询服务 | 商机阶段管理、跟进记录、方案协同 |
| **渠道销售** | 代理商/经销商管理、渠道赋能 | 快消品、设备销售 | 渠道管理、线索分发、业绩归属 |
| **私域运营** | 微信/企微运营、社群营销、内容触达 | 零售、教育、美业 | 企微接入、标签管理、自动化触达 |

#### 2.3.2 客服型组织形态

| 组织形态 | 特征 | 适用场景 | 系统支持重点 |
|----------|------|----------|--------------|
| **售前咨询** | 产品咨询、报价支持、线索转化 | 电商、软件、服务 | 会话管理、知识库、转销售 |
| **售后服务** | 问题处理、工单流转、满意度管理 | 硬件、软件、服务 | 工单管理、SLA、知识库 |
| **技术支持** | 技术问题、远程协助、升级处理 | 软件、设备、IT服务 | 工单管理、知识库、升级机制 |
| **客户成功** | 续费跟进、增购推荐、价值交付 | SaaS、订阅服务 | 客户健康度、续费提醒、经营分析 |

### 2.4 业务模式支持矩阵

#### 2.4.1 首批重点支持的业务模式

| 业务模式 | 销售模式 | 客服模式 | 典型行业 | 支持程度 |
|----------|----------|----------|----------|----------|
| **B2B SaaS** | 网销+顾问式 | 技术支持+客户成功 | 软件行业 | ★★★★★ |
| **教育培训** | 电销+顾问式 | 售前咨询+售后答疑 | 教育行业 | ★★★★★ |

| **金融服务** | 电销+渠道 | 售前咨询+售后支持 | 金融行业 | ★★★★☆ |

#### 2.4.2 暂不优先支持的业务模式

| 业务模式 | 原因 | 计划纳入阶段 |
|----------|------|--------------|
| 纯电销模式 | 需要呼叫中心集成 | P1 |
| 线下门店 | 需要POS/库存集成 | P2 |
| 分销/代理模式 | 渠道管理复杂度高 | P2 |
| 跨境电商 | 多语言、多币种、合规复杂 | P3 |

### 3.2 业务阶段详细定义

#### 3.2.1 获客阶段（线索经营）

| 维度 | 定义 |
|------|------|
| **阶段目标** | 将潜在客户信息转化为可跟进的线索，完成首次触达 |
| **参与角色** | 市场人员、销售经理、销售人员 |
| **输入** | 渠道流量、表单提交、活动报名、人工录入、API导入 |
| **输出** | 已分配的线索、首次跟进记录 |
| **关键动作** | 线索录入、线索清洗、线索分配、首次跟进 |
| **核心数据对象** | 线索(Lead)、渠道(Channel)、跟进记录(FollowUp) |
| **阶段切换条件** | 线索分配给销售人员并完成首次跟进 |
| **AI可参与任务** | 线索去重、线索评分、自动分配建议、首次触达话术推荐 |
| **人工必须决策** | 线索有效性判断、分配对象选择、跟进策略制定 |

#### 3.2.2 沟通阶段（会话管理）

| 维度 | 定义 |
|------|------|
| **阶段目标** | 建立客户沟通渠道，完成需求探索和初步意向确认 |
| **参与角色** | 销售人员、客服人员、客户 |
| **输入** | 客户咨询、主动触达、会话请求 |
| **输出** | 会话记录、客户需求记录、意向判断 |
| **关键动作** | 渠道接入、会话响应、需求探索、意向标记 |
| **核心数据对象** | 会话(Conversation)、消息(Message)、客户(Customer) |
| **阶段切换条件** | 确认客户有明确购买意向，创建商机 |
| **AI可参与任务** | 智能回复、话术推荐、需求识别、情绪分析 |
| **人工必须决策** | 需求确认、意向判断、商机创建决策 |

#### 3.2.3 谈单阶段（商机推进）

| 维度 | 定义 |
|------|------|
| **阶段目标** | 推进商机阶段，完成方案报价和商务谈判 |
| **参与角色** | 销售人员、销售经理、客户决策者 |
| **输入** | 商机信息、客户需求、竞品信息 |
| **输出** | 方案文档、报价单、合同草案 |
| **关键动作** | 需求确认、方案制定、报价沟通、商务谈判 |
| **核心数据对象** | 商机(Opportunity)、跟进记录(FollowUp)、报价(Quote) |
| **阶段切换条件** | 客户确认合作意向，进入合同签署 |
| **AI可参与任务** | 竞品分析、话术建议、成交概率预测、风险提示 |
| **人工必须决策** | 方案制定、报价策略、谈判策略、赢单/输单判断 |

#### 3.2.4 成交阶段（订单管理）

| 维度 | 定义 |
|------|------|
| **阶段目标** | 完成合同签署和订单确认，建立正式客户关系 |
| **参与角色** | 销售人员、财务人员、客户 |
| **输入** | 商机赢单、合同草案、客户信息 |
| **输出** | 签署合同、正式订单、客户档案 |
| **关键动作** | 合同签署、订单创建、客户建档、交付准备 |
| **核心数据对象** | 订单(Order)、合同(Contract)、客户(Customer) |
| **阶段切换条件** | 合同签署完成，订单确认 |
| **AI可参与任务** | 合同风险提示、客户信息补全、交付提醒 |
| **人工必须决策** | 合同条款确认、价格审批、客户建档 |

#### 3.2.5 售后阶段（工单服务）

| 维度 | 定义 |
|------|------|
| **阶段目标** | 解决客户问题，维护客户满意度，建立服务口碑 |
| **参与角色** | 客服人员、技术支持、客户 |
| **输入** | 客户问题、服务请求、投诉反馈 |
| **输出** | 问题解决方案、服务记录、满意度评价 |
| **关键动作** | 问题受理、工单创建、问题处理、服务评价 |
| **核心数据对象** | 工单(Ticket)、服务记录(ServiceRecord)、评价(Evaluation) |
| **阶段切换条件** | 问题解决，客户确认满意 |
| **AI可参与任务** | 问题分类、知识推荐、解决方案建议、满意度预测 |
| **人工必须决策** | 问题诊断、解决方案制定、复杂问题处理 |

#### 3.2.6 复购阶段（客户经营）

| 维度 | 定义 |
|------|------|
| **阶段目标** | 维护客户关系，促进续费增购，实现客户价值最大化 |
| **参与角色** | 客户成功经理、销售人员、客户 |
| **输入** | 客户使用数据、满意度数据、续费时间 |
| **输出** | 续费订单、增购订单、转介绍线索 |
| **关键动作** | 客户健康度监控、续费提醒、增购推荐、转介绍激励 |
| **核心数据对象** | 客户(Customer)、续费记录(Renewal)、转介绍(Referral) |
| **阶段切换条件** | 客户完成续费或增购，或产生转介绍 |
| **AI可参与任务** | 流失预警、续费概率预测、增购推荐、最佳触达时机 |
| **人工必须决策** | 续费策略、增购方案、客户关怀决策 |

## 四、销售与客服一体化设计

### 4.1 一体化设计原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **客户视角连续** | 客户感知不到销售与客服的边界 | 统一客户档案、统一沟通渠道、统一服务标准 |
| **数据互通共享** | 销售、客服数据实时同步 | 统一客户ID、统一会话记录、统一服务历史 |
| **角色协同无缝** | 销售、客服角色可灵活切换 | 会话转接、工单转派、协作跟进 |
| **流程闭环衔接** | 售后问题可转化为销售机会 | 工单转商机、投诉转跟进、服务转经营 |

| 销售转客服 | 客户需要技术支持 | 工单创建 + 分配 | 客户信息、问题背景 |
| 投诉升级 | 客户投诉需销售介入 | 工单升级 + 通知 | 投诉内容、客户价值 |
| 续费跟进 | 客户即将到期 | 续费任务 + 分配销售 | 使用数据、服务记录 |
| 增购推荐 | 客户有增购潜力 | 商机创建 + 分配销售 | 使用数据、客户画像 |
| 转介绍处理 | 客户推荐新客户 | 线索创建 + 标记来源 | 原客户信息、关系链 |

#### 4.4.2 核心对象定义

| 对象 | 定义 | 核心属性 | 关键关系 | 状态机 |
|------|------|----------|----------|--------|
| **渠道(Channel)** | 客户来源渠道 | 渠道名称、渠道类型、渠道状态 | 产生线索、关联会话 | 启用/停用 |
| **线索(Lead)** | 潜在客户信息 | 线索来源、联系方式、意向度 | 属于渠道、转化为客户/商机 | 5状态 |
| **客户(Customer)** | 已建立业务关系的客户企业 | 企业名称、行业、规模、客户阶段 | 拥有联系人、商机、工单 | 生命周期阶段 |
| **联系人(Contact)** | 客户企业的具体联系人 | 姓名、职位、联系方式、决策角色 | 属于客户、参与会话 | 活跃/非活跃 |
| **商机(Opportunity)** | 有明确购买意向的销售机会 | 商机名称、金额、阶段、赢率 | 属于客户、关联联系人 | 6阶段 |
| **报价(Quote)** | 商机报价单 | 报价金额、有效期、报价状态 | 属于商机、关联产品 | 草稿/已发送/已确认/已拒绝 |
| **会话(Conversation)** | 客户与客服/销售的沟通交互 | 渠道、状态、开始时间、结束时间 | 关联客户/联系人、包含消息 | 3状态 |
| **消息(Message)** | 会话中的单条沟通内容 | 内容、发送者、发送时间、消息类型 | 属于会话 | 无状态 |
| **工单(Ticket)** | 客户问题或请求的服务记录 | 标题、类型、优先级、状态 | 属于客户、关联会话 | 5状态 |
| **服务记录(ServiceRecord)** | 工单处理过程记录 | 处理内容、处理人、处理时间 | 属于工单 | 无状态 |
| **跟进记录(FollowUp)** | 销售/客服跟进记录 | 跟进内容、跟进方式、跟进时间 | 关联线索/商机/客户 | 无状态 |
| **任务(Task)** | 待办事项 | 任务内容、截止时间、负责人、状态 | 关联线索/商机/工单 | 待办/进行中/已完成/已取消 |
| **知识条目(Knowledge)** | 知识库条目 | 标题、分类、内容、状态 | 被会话/工单引用 | 草稿/已发布/已下架 |
| **AI任务(AITask)** | AI执行的任务 | 任务类型、输入、输出、状态 | 关联会话/工单 | 待执行/执行中/已完成/失败 |

#### 5.1.2 重复线索处理

| 场景 | 判定规则 | 处理策略 | 配置项 |
|------|----------|----------|--------|

| 部分重复 | 手机号相同，姓名不同 | 提示确认，人工判断是否合并 | 可配置重复判定字段 |
| 跨客户重复 | 手机号已存在于其他客户 | 提示确认，人工判断是否关联 | 可配置跨客户处理策略 |

**重复线索处理配置项：**

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 重复判定字段 | 手机号、邮箱 | 用于判定线索是否重复的字段组合 |
| 自动合并开关 | 关闭 | 是否自动合并完全重复的线索 |
| 重复提示阈值 | 1 | 相似度达到多少时提示可能重复 |
| 保留策略 | 保留最新 | 合并时保留哪条线索的信息 |

#### 5.1.3 线索回收处理

| 场景 | 触发条件 | 处理流程 | 系统动作 |
|------|----------|----------|----------|

**线索回收配置项：**

| 配置项 | 默认值 | 说明 |

| 未跟进回收天数 | 7天 | 超过多少天未跟进自动回收 |
| 无响应标记次数 | 3次 | 连续多少次无响应标记为沉睡 |
| 回收后状态 | new | 回收后线索的状态 |
| 回收通知 | 开启 | 是否通知原负责人 |

## 七、业务规则配置化设计

### 7.1 可配置规则清单

| 规则类型 | 规则名称 | 配置项 | 默认值 | 影响范围 |
|----------|----------|--------|--------|----------|
| 线索分配 | 分配方式 | 轮询/负载均衡/手动 | 轮询 | 线索管理 |
| 线索分配 | 分配范围 | 全员/团队/指定人员 | 团队 | 线索管理 |
| 线索回收 | 未跟进回收天数 | 天数 | 7天 | 线索管理 |
| 线索去重 | 重复判定字段 | 字段组合 | 手机号+姓名 | 线索管理 |
| 商机阶段 | 阶段定义 | 阶段名称、赢率 | 6阶段 | 商机管理 |
| 商机暂停 | 暂停最大天数 | 天数 | 90天 | 商机管理 |
| SLA响应 | 各优先级时限 | 时限（分钟） | 紧急15分 | 工单管理 |
| SLA解决 | 各优先级时限 | 时限（小时） | 紧急4小时 | 工单管理 |
| 会话分配 | 分配方式 | 轮询/技能组/手动 | 轮询 | 会话管理 |
| 会话超时 | 无响应自动结束 | 分钟数 | 30分钟 | 会话管理 |
| 客户沉默 | 沉默判定天数 | 天数 | 30天 | 客户管理 |
| 客户流失 | 流失判定天数 | 天数 | 90天 | 客户管理 |

### 7.2 规则配置界面需求

| 配置界面 | 配置项数量 | 权限要求 | 说明 |
|----------|------------|----------|------|
| 线索规则配置 | 5项 | 系统管理员 | 线索分配、回收、去重规则 |
| 商机规则配置 | 3项 | 系统管理员 | 商机阶段、暂停、输单规则 |
| 工单规则配置 | 4项 | 系统管理员+客服经理 | SLA、优先级、升级规则 |
| 会话规则配置 | 3项 | 系统管理员+客服经理 | 分配、超时、转接规则 |
| 客户规则配置 | 3项 | 系统管理员 | 沉默、流失、黑名单规则 |
