## 6. 跨模块协作事务（关键）

### 6.1 线索转商机（LM -> OM）
1. 锁定 lead 行（for update）。
2. 校验 `SM-lead` 可转化。
3. 创建 opportunity。
4. lead 状态改 `converted` + 关联 opportunity_id。
5. 写审计 `LEAD_CONVERT`。

### 6.2 会话转工单（CNV -> TK）
1. 校验会话状态与权限。
2. 创建 ticket 与 ticket_log。
3. 关联 source_type/source_id。
4. 推送 `ticket.created`。
5. 写审计 `CONVERSATION_CREATE_TICKET`。

## 7. 审计与日志落点
- Application Service 完成写操作后统一调用 `AuditService.record()`。
- 关键字段：`request_id/org_id/operator_id/api_id/action/target_type/target_id/result`。

## 8. 异常处理约束
- 领域异常映射为业务错误码（见 `28_API错误码清单.md`）。
- 不允许吞错后返回 `OK`。
