## 2. 分层规范
- Controller：协议层，参数校验与权限注解。
- Application Service：用例编排与事务边界。
- Domain Service：状态机与领域规则。
- Repository：数据访问与租户过滤。

## 4. 关键设计约束
- 所有状态变更必须经 Domain Service 校验合法流转。
- 所有写操作必须记录审计日志。
- 任何跨模块写操作必须显式事务边界。

## 5. 模块协作示例：会话转工单
1. CNV 模块校验会话状态（SM-conversation）。
2. 权限校验 `PERM-CNV-CREATE_TICKET`。
3. 调用 TK 模块创建工单（TABLE-TK-001）。
4. 写入 AUD 日志。
