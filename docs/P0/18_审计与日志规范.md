## 2. 审计目标
- 所有关键写操作可追溯。
- 支持问题排查、合规取证与链路复盘。

## 3. 审计落库
- 表：`TABLE-AUD-001 audit_logs`。
- 写入时机：业务写操作提交后。

## 4. 审计字段（最小必填）
`request_id/org_id/operator_id/api_id/action/target_type/target_id/result/created_at`

扩展字段：
`error_code/before_data/after_data/ip/user_agent`。

## 5. 审计触发点（P0）
- 客户创建、更新、状态变更
- 线索分配、跟进、转化
- 商机阶段推进、结果标记
- 会话接入、转接、关闭、转工单
- 工单创建与流转
- 角色权限配置
- 组织/部门变更
- 系统配置更新
- AI 任务创建与执行结果

## 6. 查询规范
- 查询接口：`API-AUD-001`。
- 查询必须按 `org_id` 隔离。
- 支持按 `request_id` 精准追踪。

## 7. 日志规范（结构化）
- 统一 JSON 日志字段：
  - `ts`,`level`,`request_id`,`org_id`,`user_id`,`api_id`,`latency_ms`,`code`,`message`
