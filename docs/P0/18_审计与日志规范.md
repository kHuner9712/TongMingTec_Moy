## 一、文档目的

本文档定义 MOY 系统的审计与日志规范，作为企业级 AI 原生客户管理系统的合规基线，用于：

1. 定义审计日志的记录范围和格式
2. 规范各类操作日志的记录要求
3. 明确数据变更的追溯能力
4. 定义 AI 执行的审计要求
5. 支撑安全合规与问题排查

## 二、审计日志体系

### 2.1 日志分类

| 日志类型 | 说明 | 存储周期 | 存储位置 |
|----------|------|----------|----------|
| 登录日志 | 用户登录/登出记录 | 180天 | audit_logs |
| 操作日志 | 关键业务操作记录 | 90天 | operation_logs |
| 数据变更日志 | 数据增删改记录 | 365天 | data_change_logs |
| AI执行日志 | AI任务执行记录 | 90天 | ai_task_logs |
| 自动化执行日志 | 自动化规则执行记录 | 90天 | automation_logs |
| 集成调用日志 | 外部系统调用记录 | 30天 | integration_logs |
| 系统日志 | 系统运行日志 | 30天 | 文件/ELK |

### 2.2 日志设计原则

| 原则 | 说明 |
|------|------|
| 完整性 | 所有关键操作必须记录，不可遗漏 |
| 不可篡改 | 日志写入后不可修改或删除 |
| 可追溯 | 每条日志可追溯到具体用户、时间、操作 |
| 结构化 | 日志格式统一，便于检索分析 |
| 分离存储 | 审计日志与业务数据分离存储 |

## 三、登录日志

### 3.1 记录场景

| 场景 | 触发条件 | 记录内容 |
|------|----------|----------|
| 登录成功 | 用户认证通过 | 用户、IP、设备、时间 |
| 登录失败 | 用户认证失败 | 用户、IP、失败原因、时间 |
| 登出 | 用户主动登出 | 用户、IP、会话时长 |
| 会话超时 | 会话自动过期 | 用户、IP、超时时间 |
| Token刷新 | Refresh Token使用 | 用户、IP、时间 |
| 强制登出 | 管理员强制下线 | 用户、操作人、原因 |

### 3.2 登录日志字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGSERIAL | 日志ID |
| org_id | BIGINT | 租户ID |
| user_id | BIGINT | 用户ID |
| username | VARCHAR(64) | 用户名 |
| login_type | VARCHAR(16) | 登录类型：password/sso/oauth |
| login_result | VARCHAR(16) | 结果：success/failure |
| failure_reason | VARCHAR(64) | 失败原因 |
| ip_address | VARCHAR(45) | IP地址（支持IPv6） |
| user_agent | VARCHAR(512) | 浏览器标识 |
| device_info | JSONB | 设备信息 |
| location | VARCHAR(128) | 地理位置 |
| session_id | VARCHAR(64) | 会话ID |
| login_at | TIMESTAMP | 登录时间 |
| logout_at | TIMESTAMP | 登出时间 |
| session_duration | INTEGER | 会话时长（秒） |

### 3.3 设备信息结构

```json
{
  "device_type": "desktop",
  "os": "Windows 11",
  "browser": "Chrome 120",
  "screen_resolution": "1920x1080",
  "device_id": "device-uuid-xxx"
}
```

### 3.4 登录异常检测

| 异常类型 | 检测规则 | 处理方式 |
|----------|----------|----------|
| 异地登录 | IP地理位置与历史不符 | 发送告警通知 |
| 异常时间 | 非工作时间登录 | 记录标记 |
| 频繁失败 | 短时间多次失败 | 账号锁定 |
| 新设备 | 首次使用的设备 | 发送验证通知 |
| 多设备同时在线 | 同账号多设备活跃 | 记录告警 |

| 操作 | 敏感级别 | 额外记录 | 告警 |

| 系统配置修改 | 高 | 配置变更内容 | 是 |
| 跨部门数据转移 | 中 | 转移原因 | 否 |

### 6.3 AI审计要求

| 要求 | 说明 |
|------|------|
| 输入追溯 | 记录完整的输入数据和上下文 |
| 输出追溯 | 记录完整的AI输出结果 |
| 模型追溯 | 记录使用的模型名称和版本 |
| 人工确认 | 高风险AI操作需人工确认 |
| 结果反馈 | 记录用户对AI结果的反馈 |

### 6.4 AI风险评估日志

| 风险类型 | 记录内容 | 处理方式 |
|----------|----------|----------|
| 低置信度 | 置信度<0.7的结果 | 标记并通知人工审核 |
| 敏感内容 | 涉及敏感信息的输出 | 记录并告警 |
| 幻觉风险 | 可能产生幻觉的场景 | 标记并限制使用 |
| 异常输出 | 输出格式异常或内容异常 | 记录并阻断 |

## 九、审计追溯能力

### 9.1 追溯维度

| 维度 | 说明 | 查询方式 |
|------|------|----------|
| 用户追溯 | 查询用户的所有操作 | user_id |
| 数据追溯 | 查询数据的变更历史 | resource_type + resource_id |
| 时间追溯 | 查询时间范围内的操作 | created_at范围 |
| 操作追溯 | 查询特定类型的操作 | module + action |
| IP追溯 | 查询特定IP的操作 | ip_address |
