## 2. 初始化目标
- 在新 `org_id` 下生成可用的最小业务运行环境。
- 初始化后可直接完成：登录、权限配置、客户/线索/会话/工单基础操作。

## 3. 输入参数（开通工单必须提供）
| 参数 | 必填 | 说明 |
| --- | --- | --- |
| org_code | 是 | 组织编码（唯一） |
| org_name | 是 | 组织名称 |
| admin_username | 是 | 管理员用户名 |
| admin_password_temp | 是 | 初始临时密码 |
| admin_email | 否 | 管理员邮箱 |
| timezone | 否 | 默认 `Asia/Shanghai` |
| locale | 否 | 默认 `zh-CN` |

## 4. 执行步骤（强制顺序）
1. 创建组织（TABLE-ORG-001）。
2. 创建默认部门（根部门，TABLE-ORG-002）。
3. 创建管理员用户（TABLE-USR-001）。
4. 初始化权限字典（TABLE-USR-003，全量 PERM）。
5. 初始化角色（TABLE-USR-002）。
6. 写入角色-权限映射（TABLE-USR-005）。
7. 写入用户-角色映射（TABLE-USR-004，管理员绑定 ROLE-ADMIN）。
8. 初始化默认渠道（TABLE-CHN-001，webchat enabled）。
9. 初始化系统配置（TABLE-SYS-001）。
10. 写入审计日志（TABLE-AUD-001，动作 `TENANT_BOOTSTRAP`）。

## 6. 可执行校验项（初始化后立即执行）
| 校验项 | 预期 |
| --- | --- |
| organizations 中 org_code 唯一 | 1 条记录 |
| 管理员用户存在且状态 active | 1 条记录 |
| 管理员角色映射完整 | ROLE-ADMIN 绑定成功 |
| permissions 数量 | 与基线权限数一致 |
| channels 默认记录 | 至少 1 条 enabled |
| system_configs 默认配置 | 4 条默认 key 全存在 |

## 7. 快速校验 SQL（示例）
```sql
select id, code, name from organizations where code = :org_code and deleted_at is null;
select id, username, status from users where org_id = :org_id and username = :admin_username and deleted_at is null;
select count(1) from permissions where org_id = :org_id;
select count(1) from role_permissions rp join roles r on rp.role_id=r.id where r.org_id=:org_id;
select config_key from system_configs where org_id=:org_id and deleted_at is null;
```

## 8. 失败回滚策略
- 若任一步骤失败，回滚当前事务。
- 已落地但整体失败时，执行租户级软删除脚本（按表依赖逆序）。
- 回滚动作必须写审计日志 `TENANT_BOOTSTRAP_ROLLBACK`。
