## 2. 性能指标与验证
| 指标 | 目标 | 验证方式 |
| --- | --- | --- |
| Web 首屏（P95） | <= 3s | Lighthouse/前端埋点 |
| API 延迟（P95） | <= 500ms | APM 指标 |
| API 延迟（P99） | <= 1000ms | APM 指标 |
| WebSocket 推送延迟（P95） | <= 1s | 事件打点差值 |
| 列表接口吞吐 | >= 100 req/s（TEST） | 压测报告 |

## 3. 可用性与可靠性
| 指标 | 目标 | 验证方式 |
| --- | --- | --- |
| 月度可用性 | >= 99.5% | 监控平台 SLA |
| 发布回滚能力 | <= 15 分钟完成回滚 | 演练记录 |
| 数据备份 | 每日一次 + 保留 7 天 | 备份任务日志 |
| 关键链路故障恢复 | <= 30 分钟 | 故障演练 |

## 4. 安全与合规
| 项 | 目标 | 验证方式 |
| --- | --- | --- |
| 鉴权 | JWT + 过期控制 | 安全测试 |
| 权限控制 | RBAC + 数据范围 | 权限自动化用例 |
| 租户隔离 | 全链路 org_id | 跨租户测试 |
| 审计留痕 | 关键写操作 100% 留痕 | 审计抽样 |
| 敏感信息脱敏 | AI 入参与日志脱敏 | 代码审查 + 用例 |

## 5. 可维护性与可观测性
| 指标 | 目标 |
| --- | --- |
| 结构化日志覆盖 | API/WS 100% |
| request_id 贯通 | 网关 -> API -> WS -> 审计 |
| 错误码标准化 | 100% 使用 `28_API错误码清单` |
| 告警可达率 | 100% 告警可通知责任人 |

## 6. 容量规划（P0）
- 同时在线用户：1000。
- 并发会话：300。
- 日消息量：50 万。
- 日审计日志量：100 万条。
- 数据保留：
  - 业务主表：长期保留
  - 审计日志：至少 180 天

## 7. 验收准入
- 所有指标在 TEST 压测与联调中达到目标。
- 任何未达标项必须有降级方案并经评审批准。
