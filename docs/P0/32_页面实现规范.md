## 一、文档目的

本文档定义 MOY 系统的页面实现规范，确保：

1. 页面结构统一、可维护
2. 页面与模块、接口、权限映射清晰
3. 页面布局标准化
4. 交互行为一致
5. 前端开发可直接基于规范实现

### 7.4 排序规范

| 规范项 | 要求 |
|--------|------|

| 排序指示 | 使用 Ant Design 默认排序图标 |
| 多列排序 | 不支持，后排序覆盖前排序 |

### 7.5 批量操作规范

## 九、状态展示规范

### 9.1 状态标签颜色规范

| 状态类型 | 状态值 | 颜色 |
|----------|--------|------|
| 新建/初始 | new, initial, pending | blue |
| 进行中 | following, processing, active | processing |
| 已完成 | converted, won, completed, closed | success |
| 已失败/无效 | invalid, lost, failed | error |
| 已分配 | assigned | cyan |
| 已解决 | resolved | cyan |
| 已归档 | archived | default |

### 9.2 状态流转交互规范

```tsx
// 状态变更下拉组件
<Dropdown
  menu={{
    items: availableTransitions.map(t => ({
      key: t.to,
      label: t.label,
      onClick: () => handleStatusChange(t.to),
    })),
  }}
>
  <Button>
    <StatusTag status={currentStatus} />
    <DownOutlined />
  </Button>
</Dropdown>

// 状态变更确认
const handleStatusChange = (newStatus: string) => {
  if (isDangerousTransition(currentStatus, newStatus)) {
    confirmStatusChange(
      statusTextMap[currentStatus],
      statusTextMap[newStatus],
      () => executeStatusChange(newStatus)
    );
  } else {
    executeStatusChange(newStatus);
  }
};
```

## 十、危险操作规范

### 10.1 危险操作定义

| 操作类型 | 危险级别 | 确认方式 |
|----------|----------|----------|
| 删除单个 | 高 | Popconfirm |
| 批量删除 | 高 | Modal + 输入确认 |
| 状态回退 | 中 | Popconfirm |
| 批量分配 | 中 | Modal |
| 数据导出 | 低 | 无需确认 |

### 10.2 危险操作确认文案

| 操作 | 标题 | 内容 |
|------|------|------|

| 批量删除 | 批量删除确认 | 确定要删除选中的 {count} 条数据吗？此操作不可恢复。 |
| 状态回退 | 状态变更确认 | 确定要将状态从"{from}"变更为"{to}"吗？ |
