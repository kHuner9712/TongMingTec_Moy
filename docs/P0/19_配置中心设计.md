## 一、文档目的

本文档定义 MOY 系统的配置中心设计，作为企业级 AI 原生客户管理系统的配置管理基线，用于：

1. 定义系统可配置项的范围和结构
2. 规范配置项的管理和维护方式
3. 支撑多租户个性化配置需求
4. 为配置管理功能开发提供设计依据

---

## 二、配置中心架构

### 2.1 配置层级

```
全局配置（平台级）
├── 系统默认配置
│   └── 不可修改的默认值
├── 租户配置（组织级）
│   ├── 覆盖系统默认配置
│   └── 租户个性化配置
└── 用户配置（用户级）
    └── 用户偏好设置
```

### 2.2 配置分类

| 分类 | 说明 | 管理权限 |
|------|------|----------|
| 系统配置 | 系统运行参数 | 平台管理员 |
| 业务配置 | 业务规则参数 | 租户管理员 |
| 字典配置 | 枚举值、选项列表 | 租户管理员 |
| 集成配置 | 外部系统对接参数 | 租户管理员 |
| AI配置 | AI模型相关参数 | AI运营管理员 |

### 2.3 配置优先级

| 优先级 | 配置来源 | 说明 |
|--------|----------|------|
| 1（最高） | 用户配置 | 用户个人偏好 |
| 2 | 租户配置 | 租户级覆盖 |
| 3 | 系统默认 | 全局默认值 |

| rule_name | VARCHAR(64) | 规则名称 |
| rule_type | VARCHAR(32) | 规则类型 |
| conditions | JSONB | 触发条件 |
| action | VARCHAR(16) | 处理动作 |
| action_config | JSONB | 动作配置 |
| status | VARCHAR(16) | 状态 |

#### 12.2.2 风控规则示例

```json
{
  "rule_name": "频繁请求限制",
  "rule_type": "rate_limit",
  "conditions": {
    "window_seconds": 60,
    "threshold": 100
  },
  "action": "block",
  "action_config": {
    "block_duration_seconds": 300,
    "message": "请求过于频繁，请稍后再试"
  }
}
```
