# MOY 配置中心设计

## 1. 文档元信息
| 属性 | 内容 |
| --- | --- |
| 文档编号 | MOY_CONFIG_001 |
| 文档版本 | v3.0 |
| 文档状态 | 已确认（实现级冻结） |
| 日期 | 2026-04-05 |

## 2. 目标
- 为 P0 提供可审计、可版本控制的系统级配置能力。

## 3. 配置存储
- 表：`TABLE-SYS-001 system_configs`。
- 主键约束：`org_id + config_key` 唯一。
- 写入必须带 `version` 做并发控制。

## 4. P0 标准配置项
| config_key | value_type | 默认值 | 说明 |
| --- | --- | --- | --- |
| conversation.timeout_minutes | number | 30 | 会话超时自动关闭阈值 |
| ticket.sla_hours | number | 24 | 工单 SLA 小时 |
| ai.timeout_ms | number | 8000 | AI 调用超时 |
| list.default_page_size | number | 20 | 列表默认分页 |

## 5. 配置变更约束
- 配置不得引入新业务状态枚举。
- 配置变更必须写审计（`SYS_CONFIG_UPDATE`）。
- 敏感配置（`is_secret=true`）前端只显示掩码。

## 6. API 对应
- 查询：`API-SYS-002`
- 更新：`API-SYS-003`
