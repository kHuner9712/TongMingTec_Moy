## 二、高优先级问题（必须修改）

### 问题 1：线索状态枚举定义冲突

| 项目 | 内容 |
|------|------|
| **问题 ID** | ISSUE-001 |
| **严重程度** | 高优先级 |
| **冲突类型** | 数据对象定义冲突 |

**冲突描述：**

| 文档 | 状态定义 |
|------|----------|
| PRD（6.2 线索字段） | 未跟进、跟进中、已转化、已失效（4个状态） |
| DBD（7.3 线索表） | new、assigned、following、converted、invalid（5个状态） |
| API（线索接口） | new、assigned、following、converted、invalid（5个状态） |

**影响：**
- DBD 和 API 比 PRD 增加了 `new`（新线索）和 `assigned`（已分配）两个状态
- 这导致线索状态流转设计在文档间不一致
- 开发人员可能按照 DBD 实现，但产品验收可能按照 PRD 执行

**修订建议：**

| 文档 | 修订内容 | 章节 |
|------|----------|------|
| PRD | 更新线索状态为：new、assigned、following、converted、invalid | 4.2.2 功能需求 / 7.2 线索字段 |
| DBD | 保持现有设计，与 PRD 对齐 | 7.3 线索表状态枚举 |
| API | 保持现有设计，与 PRD 对齐 | 线索接口状态枚举 |

### 问题 3：知识库模块优先级不一致

| 项目 | 内容 |
|------|------|
| **问题 ID** | ISSUE-003 |
| **严重程度** | 中优先级 |
| **冲突类型** | 模块优先级冲突 |

**冲突描述：**

| 文档 | 知识库定位 |
|------|----------|
| PRD（1.1 MVP 范围） | P1（知识库基础版） |
| HLD（6.2.6 知识库模块） | P1 |

**修订建议：**

| 文档 | 修订内容 | 章节 |

| PRD | 明确知识库中哪些功能是 P0（检索、详情、AI问答），哪些是 P1（管理、分类） | 1.1 MVP 范围说明 |

### 问题 7：部分验收标准不可验证

| 项目 | 内容 |
|------|------|
| **问题 ID** | ISSUE-007 |
| **严重程度** | 中优先级 |
| **冲突类型** | 验收标准缺失或不可验证 |

**冲突描述：**
- PRD（13 验收标准）中多为定性描述，如"功能正常"、"展示正常"
