## 2. 文档目的
- 定义业务目标、业务边界与优先级。
- 明确当前正式开发范围与暂不纳入范围。
- 为 PRD 与 RTM 提供唯一业务输入。

## 3. 业务目标
| 业务目标ID | 目标描述 | 验证口径 |
| --- | --- | --- |
| BUS-OBJ-001 | 提升线索转化效率 | 线索到商机转化率提升 |
| BUS-OBJ-002 | 提升销售推进效率 | 商机阶段推进周期缩短 |
| BUS-OBJ-003 | 提升客户服务时效 | 首次响应时长与解决时长下降 |
| BUS-OBJ-004 | 建立客户经营闭环 | 获客-沟通-谈单-成交-售后链路可追踪 |
| BUS-OBJ-005 | 降低执行偏差 | 同一需求在前后端/测试口径一致 |

## 4. 当前正式开发范围（P0）

### 4.1 P0 业务模块
- 客户管理（CM）
- 线索管理（LM）
- 商机管理（OM）
- 会话中心（CNV）
- 工单管理（TK）
- 任务管理（TSK）
- 认证与权限（AUTH/ORG/USR）
- 通知中心（NTF）
- 渠道管理（CHN）
- AI 智能回复（AI，P0 轻量）
- 审计日志与系统配置（AUD/SYS）

### 4.2 P0 业务能力边界
- 允许：围绕 P0 模块完成全量可交付开发。
- 不允许：引入 P1/P2 模块作为 P0 前置依赖。

## 5. 不进入当前正式开发范围

### 5.1 P1（暂不纳入）
- 知识库管理（KB）
- 数据看板（DASH）
- 自动化规则（AUTO）
- AI 工作台高级能力（AIOPS）

### 5.2 P2（保留规划）
- 合同、订单、发票、续费
- 呼叫中心
- 移动端
