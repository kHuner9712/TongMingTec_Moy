## 2. P0 范围与边界

### 2.1 P0 内能力
- `REQ-AI-001`：智能回复生成（`API-AI-001`）。
- `REQ-AI-002`：AI 任务状态查询（`API-AI-002`）。

### 2.2 明确不纳入
- 知识库问答（qa）
- 自动决策业务终态
- AI 工作台高级运营能力

### 2.3 强制边界
- AI 输出只能作为建议，不可直接驱动 `won/lost`、工单关闭等关键终态。

## 3. AI Provider 抽象层设计

### 3.1 抽象接口（后端）
```ts
interface AiProvider {
  name: string;
  generateSmartReply(input: SmartReplyInput, options: ProviderOptions): Promise<SmartReplyOutput>;
}
```

### 3.2 SmartReplyInput
- `org_id`
- `conversation_context`（最近 N 条消息）
- `latest_user_message`
- `language`
- `instruction`
- `prompt_template_id`
- `prompt_vars`

### 3.3 SmartReplyOutput
- `reply_text`
- `confidence`（0~1）
- `safety_flags`（数组）
- `usage.input_tokens`
- `usage.output_tokens`
- `provider_raw`（可选）

### 3.4 Provider 切换策略
- 主 Provider：`AI_PROVIDER_PRIMARY`。
- 备 Provider：`AI_PROVIDER_FALLBACK`（可选）。
- 主 Provider 熔断期间自动切换备 Provider；若均不可用则降级人工。

## 4. 调用时机（P0）
- 仅在 `PAGE-CNV-002` 用户主动点击“智能回复”时触发。
- 自动触发（定时/规则触发）不在 P0。

## 5. Prompt 模板管理

### 5.1 存储方式
- 使用 `TABLE-SYS-001 system_configs` 存储模板，不新增 P0 表。
- key 约定：
  - `ai.prompt.smart_reply.v1.system`
  - `ai.prompt.smart_reply.v1.user`

### 5.2 版本管理
- 模板 ID：`smart_reply_v{n}`。
- 发布模板必须走配置变更审计（`SYS_CONFIG_UPDATE`）。

### 5.3 变量来源
| 变量 | 来源 |
| --- | --- |
| `{{conversation_status}}` | conversations.status |
| `{{customer_name}}` | customers.name |
| `{{recent_messages}}` | conversation_messages（最近 20 条） |
| `{{agent_instruction}}` | API-AI-001.instruction |
| `{{language}}` | API-AI-001.language |

## 6. 脱敏与安全规则

### 6.1 脱敏字段
- 手机号：中间 4 位脱敏
- 邮箱：用户名部分脱敏
- 身份证/证件号：中间位脱敏
- 密钥/令牌：全量打码

### 6.2 脱敏执行点
1. 写入 `prompt_vars` 前。
2. 发送 Provider 前。
3. 记录审计与日志前。

### 6.3 输出安全
- 若命中敏感策略 `safety_flags`，前端仅显示“建议人工回复”。

## 7. 超时、重试、熔断、限流
| 项 | 策略 |
| --- | --- |
| 超时 | 8s（可由 `ai.timeout_ms` 配置） |
| 重试 | 最多 1 次，仅网络/5xx 可重试 |
| 熔断 | 连续 5 次失败触发，30s 后半开 |
| 限流 | `org_id + user_id` 维度，60 次/分钟 |

失败后错误码：
- 超时：`AI_TASK_TIMEOUT`
- 熔断：`AI_PROVIDER_CIRCUIT_OPEN`
- 限流：`RATE_LIMITED`

## 8. 结构化输出约束

### 8.1 目标 Schema
```json
{
  "reply_text": "string",
  "confidence": 0.82,
  "safety_flags": ["none"],
  "reasoning_brief": "string"
}
```

### 8.2 校验
- `reply_text` 必填，长度 1~4000。
- `confidence` 范围 0~1。
- 校验失败返回 `AI_OUTPUT_INVALID`，任务标记 `failed`。

## 9. 落库规则（TABLE-AI-001）
- 创建任务时：`status=pending`。
- 开始执行：`status=running`，写入 `started_at/provider/model/prompt_template_id`。
- 成功：`status=succeeded`，写入 `output_text/output_payload/token_input/token_output/cost_amount/finished_at`。
- 失败：`status=failed`，写入 `error_code/error_message/finished_at`。

## 10. 人工接管与前端降级

### 10.1 人工接管规则
- AI 失败/超时/安全拦截时，消息输入框保持可用。
- 操作员可手动编辑建议后发送。

### 10.2 UI 行为
| 场景 | UI 行为 |
| --- | --- |
| AI 任务 pending/running | 显示 loading 状态，不锁定输入框 |
| AI succeeded | 将 `reply_text` 填入草稿区，用户确认后发送 |
| AI failed/timeout | toast 提示 + 保留人工输入 |

## 11. 成本与调用日志字段
- 必填记录：
  - `request_id`
  - `org_id`
  - `operator_id`
  - `provider`
  - `model`
  - `token_input`
  - `token_output`
  - `cost_amount`
  - `latency_ms`
  - `result_status`

日志落点：
- 业务表：`TABLE-AI-001`
- 审计表：`TABLE-AUD-001`（动作 `AI_TASK_CREATE`）

## 12. 权限与租户校验
- 调用权限：`PERM-AI-EXECUTE`。
- 会话与 AI 任务查询都必须校验 `org_id` 与数据范围。

## 13. 与业务终态隔离规则
- AI 不得自动调用 `API-OM-006`、`API-TK-007`、`API-CNV-007` 等关键终态接口。
- 关键终态只能由人工触发业务 API。

## 14. 测试要求
- Provider 正常/失败/超时分支。
- 重试与熔断触发。
- 输出 schema 校验失败分支。
- 降级 UI 可人工接管。
- 成本日志与审计字段完整性。
