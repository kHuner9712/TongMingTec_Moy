## 一、文档目的

本文档作为 MOY 项目的**AI编码开发总入口**，为 Trae 提供精准、结构化的开发指导，确保：

1. **可直接指导全量开发**：每个模块的开发顺序、前置条件、依赖关系清晰
2. **可直接指导前后端联调**：接口、页面、数据表映射完整
3. **可直接指导数据库实现**：表结构、字段、索引、约束明确
4. **可直接指导测试设计与验收**：测试点、验收项、冒烟测试顺序定义
5. **可直接指导 Trae 按模块生成代码**：每个模块的输入文档、输出物、禁止事项明确

**核心原则：** 让 Trae 可以按模块精准推进，无需人工补充上下文。

### 2.3 并行与串行关系

| 关系类型 | 模块组合 | 说明 |
|----------|----------|------|
| **必须串行** | 数据库初始化 → 所有业务模块 | 数据库表必须先创建 |
| **必须串行** | 认证授权 → 所有业务模块 | 业务模块依赖用户/权限体系 |
| **必须串行** | 客户管理 → 线索管理 | 线索转化依赖客户 |
| **必须串行** | 客户管理 → 商机管理 | 商机关联客户 |
| **必须串行** | 线索管理 → 商机管理 | 线索转化创建商机 |
| **必须串行** | 客户管理 → 会话管理 | 会话关联客户 |
| **必须串行** | 客户管理 → 工单管理 | 工单关联客户 |
| **可并行** | 会话管理、工单管理 | 无直接依赖，可并行 |
| **可并行** | 知识库、数据看板、任务、通知、渠道、自动化、AI | 无直接依赖，可并行 |
