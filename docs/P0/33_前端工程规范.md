## 2. 路由规范
- 统一路由参数写法：`/path/{id}`（在代码中对应 `:id` 路由定义，但文档口径统一 `{id}`）。
- 根路由 `/` 必须重定向到 `/dashboard`。
- 页面路由必须来自 PAGE 清单，禁止自发新增 P1 页面。

## 3. API Client 规范
- API ID 与前端 service 一一对应。
- 所有请求走统一 `httpClient`，携带 `request_id`。
- 错误处理统一读取 `code/message/request_id`。

## 4. 类型与状态规范
- 状态枚举类型必须来自 `SM-*` 定义。
- 禁止在组件内重新声明冲突枚举。

## 5. 权限守卫规范
- 页面级守卫：按 PAGE 对应权限。
- 动作级守卫：按 API 对应权限。
- 会话模块必须拆分动作权限：`SEND/ACCEPT/TRANSFER/CLOSE`。

## 6. WebSocket 接入规范
- 仅消费事件，不直接写业务状态。
- 事件名必须来自 `29_WebSocket协议规范.md`。

## 7. 测试要求
- 每个 PAGE 至少 1 条渲染测试。
- 每个高风险动作至少 1 条权限拦截测试。
