## 一、产品概述

### 1.1 产品定位

| 维度     | 定义                                               |
| -------- | -------------------------------------------------- |
| 产品名称 | MOY（墨言）- Mate Of You                           |
| 产品定位 | 企业级 AI 原生客户经营系统                         |
| 核心价值 | 不是"会聊天的 AI 工具"，而是"可交付经营结果的系统" |
| 目标用户 | 销售型公司、客服团队                               |
| 终端形态 | Web 端（SaaS），首期不做 PC 客户端和移动端         |
| 核心链路 | 获客 → 沟通 → 谈单 → 成交 → 售后 → 复购/转介绍     |

### 1.2 产品目标

| 目标类型 | 目标描述                      | 衡量指标                   |
| -------- | ----------------------------- | -------------------------- |
| 业务目标 | 帮助 B 端客户降本、提效、增收 | 客户续费率 > 80%，NPS > 40 |
| 产品目标 | 实现销售与客服一体化经营      | 核心链路闭环率 100%        |
| 用户目标 | 提升销售/客服工作效率         | 人效提升 30%+              |

### 1.3 版本范围定义

#### 1.3.1 范围术语

| 术语           | 定义                   | 包含内容                                         |
| -------------- | ---------------------- | ------------------------------------------------ |
| **首期完整版** | 第一个可交付的生产版本 | P0全部功能 + P1全部功能                          |
| **P0**         | 核心业务功能           | 客户管理、线索管理、会话管理、商机管理、工单管理 |
| **P1**         | 增强业务功能           | 知识库管理、数据看板                             |

## 四、模块需求详述

### 4.1 登录/认证模块

#### 4.1.1 模块概述

| 维度     | 定义                             |
| -------- | -------------------------------- |
| 模块目标 | 提供安全的用户身份认证和会话管理 |
| 用户角色 | 全部用户                         |
| 优先级   | P0                               |

#### 4.1.2 功能需求

| 需求ID       | 功能名称       | 功能描述                         | 优先级 |
| ------------ | -------------- | -------------------------------- | ------ |
| REQ-AUTH-001 | 账号登录       | 支持手机号/邮箱 + 密码登录       | P0     |
| REQ-AUTH-002 | 密码找回       | 支持通过手机号/邮箱重置密码      | P0     |
| REQ-AUTH-003 | 会话管理       | 登录后创建会话，支持单点登录控制 | P0     |
| REQ-AUTH-004 | 登出           | 支持用户主动登出，清除会话       | P0     |
| REQ-AUTH-005 | 登录日志       | 记录登录时间、IP、设备信息       | P0     |
| REQ-AUTH-006 | 密码修改       | 支持用户修改自己的密码           | P0     |
| REQ-AUTH-007 | 多设备登录控制 | 可配置是否允许多设备同时登录     | P1     |

#### 4.1.3 业务规则

| 规则ID        | 规则描述                              | 优先级 |
| ------------- | ------------------------------------- | ------ |
| RULE-AUTH-001 | 密码长度不少于8位，必须包含字母和数字 | P0     |
| RULE-AUTH-002 | 连续5次登录失败，账号锁定30分钟       | P0     |
| RULE-AUTH-003 | 会话有效期默认24小时，可配置          | P0     |
| RULE-AUTH-004 | 登录成功后刷新会话有效期              | P0     |
| RULE-AUTH-005 | 密码加密存储，使用bcrypt算法          | P0     |

#### 4.1.4 页面元素

**登录页：**

- 页面目标：完成用户身份认证
- 页面元素：
  - 登录表单：账号输入框、密码输入框、记住我复选框、登录按钮
  - 辅助链接：忘记密码、注册账号（如开放注册）
  - 错误提示：账号不存在、密码错误、账号锁定

**密码找回页：**

- 页面目标：帮助用户重置密码
- 页面元素：
  - 找回表单：账号输入框、验证码输入框、发送验证码按钮
  - 重置表单：新密码输入框、确认密码输入框、提交按钮

#### 4.1.5 验收标准

| 验收项   | 前置条件        | 操作步骤                   | 期望结果             |
| -------- | --------------- | -------------------------- | -------------------- |
| 正常登录 | 已注册账号      | 输入正确账号密码，点击登录 | 登录成功，跳转工作台 |
| 密码错误 | 已注册账号      | 输入错误密码，点击登录     | 提示密码错误         |
| 账号锁定 | 连续5次密码错误 | 第6次尝试登录              | 提示账号已锁定       |
| 密码找回 | 已注册账号      | 点击忘记密码，输入账号     | 发送验证码成功       |

### 6.3 补充状态机定义

#### 6.3.1 用户状态机

| 状态（中文） | 状态（英文） | 说明                           | 允许流转目标     |
| ------------ | ------------ | ------------------------------ | ---------------- |
| 正常         | active       | 用户正常使用                   | inactive, locked |
| 停用         | inactive     | 用户被停用                     | active           |
| 锁定         | locked       | 用户被锁定（密码错误次数过多） | active           |

**流转规则：**

- active → inactive：管理员停用用户
- active → locked：连续登录失败超过阈值
- inactive → active：管理员启用用户
- locked → active：管理员解锁或用户重置密码

#### 6.3.2 组织状态机

| 状态（中文） | 状态（英文） | 说明                 | 允许流转目标      |
| ------------ | ------------ | -------------------- | ----------------- |
| 正常         | active       | 组织正常运营         | suspended, closed |
| 暂停         | suspended    | 组织被暂停（欠费等） | active, closed    |
| 关闭         | closed       | 组织已关闭           | 终态              |

**流转规则：**

- active → suspended：欠费或违规
- active → closed：组织注销
- suspended → active：恢复正常
- suspended → closed：组织注销

#### 6.3.3 部门状态机

| 状态（中文） | 状态（英文） | 说明                   | 允许流转目标 |
| ------------ | ------------ | ---------------------- | ------------ |
| 正常         | active       | 部门正常运营           | archived     |
| 归档         | archived     | 部门已归档（不再使用） | active       |

**流转规则：**

- active → archived：部门归档（需先移除或转移部门成员）
- archived → active：恢复部门

#### 6.3.4 自动化规则状态机

| 状态（中文） | 状态（英文） | 说明       | 允许流转目标 |
| ------------ | ------------ | ---------- | ------------ |
| 启用         | active       | 规则生效中 | inactive     |
| 停用         | inactive     | 规则已停用 | active       |

**流转规则：**

- active → inactive：管理员停用规则
- inactive → active：管理员启用规则

#### 6.3.5 知识库条目状态机

| 状态（中文） | 状态（英文） | 说明                 | 允许流转目标        |
| ------------ | ------------ | -------------------- | ------------------- |
| 草稿         | draft        | 草稿状态，未发布     | published, archived |
| 已发布       | published    | 已发布状态，可被检索 | draft, archived     |
| 已归档       | archived     | 已归档状态，不可检索 | draft               |

**流转规则：**

- draft → published：发布知识
- draft → archived：归档草稿
- published → draft：下架重新编辑
- published → archived：归档知识
- archived → draft：恢复为草稿
