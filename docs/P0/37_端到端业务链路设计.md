## 一、文档目的

本文档作为 MOY 项目的**端到端业务链路设计基线**，用于：

1. 完整定义六大业务链路的端到端流程
2. 明确每个链路的业务、系统、数据、状态、页面、接口、权限、审计、测试要求
3. 补齐成交链路和复购/转介绍链路的设计缺口
4. 定义系统边界与未来扩展蓝图
5. 为全量业务实现开发提供完整指导

**核心原则：** 每个链路都必须形成闭环，即使首期不实现的功能也要明确系统边界和扩展预留。

### 3.3 业务场景清单

| 场景编号 | 场景名称 | 触发条件 | 处理流程 | 状态变更 |
|----------|----------|----------|----------|----------|
| LC-001 | 手动录入线索 | 用户手动填写线索表单 | 创建线索 → 重复检测 → 分配 | new → assigned |
| LC-002 | 批量导入线索 | 上传CSV/Excel文件 | 解析 → 去重 → 批量创建 → 批量分配 | new → assigned |
| LC-003 | 渠道自动接入 | 渠道API接收线索 | 创建线索 → 自动分配 | new → assigned |
| LC-004 | 重复线索检测 | 手机号/邮箱匹配 | 提示确认 → 合并/关联/跳过 | - |
| LC-005 | 线索自动分配 | 新线索进入分配池 | 按规则分配（轮询/负载均衡） | new → assigned |
| LC-006 | 线索手动分配 | 管理员选择负责人 | 分配给指定销售人员 | new → assigned |
| LC-007 | 首次跟进 | 销售添加跟进记录 | 记录跟进内容 → 更新状态 | assigned → following |
| LC-008 | 线索转化 | 确认购买意向 | 创建商机 → 创建/关联客户 | following → converted |
| LC-009 | 线索标记无效 | 判定为无效线索 | 填写无效原因 → 归档 | * → invalid |
| LC-010 | 线索回收 | 超过N天未跟进 | 回收到公海 → 重新分配 | following → new |
| LC-011 | 线索领取 | 销售从公海领取 | 领取线索 → 成为负责人 | new → assigned |

### 3.4 数据流转

| 步骤 | 输入数据 | 输出数据 | 数据对象变更 |
|------|----------|----------|--------------|
| 线索录入 | 姓名、手机号、来源等 | 线索记录 | 创建 Lead |
| 重复检测 | 手机号/邮箱 | 重复结果 | 查询 Lead |
| 线索分配 | 负责人ID | 分配记录 | Lead.owner_id 更新 |
| 首次跟进 | 跟进内容 | 跟进记录 | 创建 FollowUp，Lead.status 更新 |
| 线索转化 | 商机信息 | 商机记录、客户记录 | 创建 Opportunity、Customer，Lead.status=converted |
| 线索失效 | 失效原因 | 失效记录 | Lead.status=invalid |
