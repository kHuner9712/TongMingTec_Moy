## 2. P0 链路总览
- 获客：线索进入与分配
- 沟通：会话接入与消息处理
- 谈单：商机推进与结果标记
- 成交：P0 仅保留商机结果，不实现合同订单
- 售后：工单闭环
- 复购：P0 仅保留客户状态经营，不实现商业化流程

## 3. 链路映射

### 3.1 获客链路
- REQ-LM-001 -> API-LM-002 -> TABLE-LM-001
- REQ-LM-002 -> API-LM-003 -> TABLE-LM-001

### 3.2 沟通链路
- REQ-CNV-001 -> API-CNV-001/002 -> TABLE-CNV-001
- REQ-CNV-002 -> API-CNV-004 -> TABLE-CNV-002
- REQ-CNV-003 -> API-CNV-005/006 -> SM-conversation

### 3.3 谈单链路
- REQ-OM-002 -> API-OM-005 -> TABLE-OM-002
- REQ-OM-003 -> API-OM-006 -> `won/lost`

### 3.4 售后链路
- REQ-CNV-005 -> API-CNV-008 -> TABLE-TK-001
- REQ-TK-003 -> API-TK-006 -> SM-ticket

## 4. 边界声明
- 合同/订单/发票/续费均为 P2 规划，不进入 P0。
- 看板、知识库、自动化规则均为 P1 规划，不进入 P0。

## 5. 测试关注点
- 线索转商机数据一致性。
- 会话转工单跨模块事务一致性。
- 工单状态机合法流转。
