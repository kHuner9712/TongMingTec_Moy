## 一、文档目的

本文档定义 MOY 系统中所有状态机的实现规范，确保：

1. 状态流转逻辑统一、可追溯
2. 前后端状态定义一致
3. 状态变更可审计
4. 防止非法状态流转
5. 支持状态变更通知

## 三、状态机详细定义

### 3.1 线索状态机

#### 3.1.1 状态定义

| 状态值 | 中文名 | 说明 | 数据库值 |
|--------|--------|------|----------|
| new | 新线索 | 刚创建，未分配 | `new` |
| assigned | 已分配 | 已分配给销售人员 | `assigned` |
| following | 跟进中 | 销售正在跟进 | `following` |
| converted | 已转化 | 转化为商机或客户 | `converted` |
| invalid | 无效 | 线索无效 | `invalid` |

#### 3.1.2 状态流转矩阵

| 当前状态 | 目标状态 | 触发事件 | 权限要求 | 业务规则 |
|----------|----------|----------|----------|----------|
| new | assigned | 分配线索 | admin, sales_manager | 必须指定负责人 |
| new | invalid | 标记无效 | admin, sales_manager, sales_rep | 需填写无效原因 |
| assigned | following | 开始跟进 | assigned_user | - |
| assigned | invalid | 标记无效 | admin, sales_manager, assigned_user | 需填写无效原因 |
| following | converted | 转化成功 | assigned_user | 必须关联商机或客户 |
| following | invalid | 标记无效 | admin, sales_manager, assigned_user | 需填写无效原因 |
| converted | - | 终态 | - | 不可变更 |
| invalid | - | 终态 | - | 不可变更 |

#### 3.1.3 实现代码

```typescript
// 状态机定义
const LEAD_STATE_MACHINE = {
  entity: 'lead',
  states: ['new', 'assigned', 'following', 'converted', 'invalid'],
  initialState: 'new',
  finalStates: ['converted', 'invalid'],
  transitions: [
    { from: 'new', to: 'assigned', event: 'assign', requiredRoles: ['admin', 'sales_manager'] },
    { from: 'new', to: 'invalid', event: 'mark_invalid', requiredRoles: ['admin', 'sales_manager', 'sales_rep'] },
    { from: 'assigned', to: 'following', event: 'start_follow', requiredRoles: ['self'] },
    { from: 'assigned', to: 'invalid', event: 'mark_invalid', requiredRoles: ['admin', 'sales_manager', 'self'] },
    { from: 'following', to: 'converted', event: 'convert', requiredRoles: ['self'] },
    { from: 'following', to: 'invalid', event: 'mark_invalid', requiredRoles: ['admin', 'sales_manager', 'self'] },
  ],
};

// 状态流转校验
function validateTransition(entity: string, currentState: string, targetState: string, user: User): boolean {
  const machine = getStateMachine(entity);
  const transition = machine.transitions.find(t => t.from === currentState && t.to === targetState);
  
  if (!transition) {
    throw new InvalidTransitionError(`Invalid transition from ${currentState} to ${targetState}`);
  }
  
  if (transition.requiredRoles && !hasRequiredRole(user, transition.requiredRoles)) {
    throw new PermissionDeniedError('Insufficient permissions for this transition');
  }
  
  return true;
}
```

CREATE TABLE status_change_logs (
    id BIGSERIAL PRIMARY KEY,
    org_id BIGINT NOT NULL,
    entity_type VARCHAR(50) NOT NULL,

### 4.2 后端实现

#### 4.2.1 状态机服务接口

```typescript
interface StateMachineService {
  getStates(entity: string): string[];
  getTransitions(entity: string): Transition[];
  canTransition(entity: string, currentState: string, targetState: string, user: User): boolean;
  transition(entity: string, entityId: bigint, targetState: string, user: User, reason?: string): Promise<void>;
  getStatusHistory(entity: string, entityId: bigint): Promise<StatusChangeLog[]>;
}
```

#### 4.2.2 状态变更装饰器

```typescript
@StateMachineTransition('lead', 'assign')
async assignLead(leadId: bigint, assigneeId: bigint, user: User) {
  const lead = await this.leadRepository.findById(leadId);
  
  await this.stateMachineService.transition('lead', leadId, 'assigned', user, {
    event: 'assign',
    metadata: { assigneeId },
  });
  
  lead.assigneeId = assigneeId;
  lead.status = 'assigned';
  await this.leadRepository.save(lead);
  
  await this.eventBus.publish(new LeadAssignedEvent(leadId, assigneeId));
}
```

#### 4.2.3 状态变更事件

```typescript
@EventsHandler(LeadStatusChangedEvent)
export class LeadStatusChangedHandler {
  async handle(event: LeadStatusChangedEvent) {
    switch (event.toStatus) {
      case 'assigned':
        await this.notificationService.sendAssignmentNotification(event.leadId, event.assigneeId);
        break;
      case 'converted':
        await this.analyticsService.trackConversion(event.leadId);
        break;
      case 'invalid':
        await this.analyticsService.trackInvalidation(event.leadId, event.reason);
        break;
    }
  }
}
```

#### 4.3.2 状态流转组件

```typescript
interface StatusTransitionProps {
  entity: string;
  currentStatus: string;
  onTransition: (targetStatus: string, reason?: string) => Promise<void>;
  disabled?: boolean;
}

export function StatusTransition({ entity, currentStatus, onTransition, disabled }: StatusTransitionProps) {
  const machine = useStateMachine(entity);
  const availableTransitions = machine.getAvailableTransitions(currentStatus);
  
  return (
    <Dropdown>
      <DropdownButton disabled={disabled || availableTransitions.length === 0}>
        变更状态
      </DropdownButton>
      <DropdownMenu>
        {availableTransitions.map(t => (
          <DropdownItem key={t.to} onClick={() => onTransition(t.to)}>
            {t.label}
          </DropdownItem>
        ))}
      </DropdownMenu>
    </Dropdown>
  );
}
```

## 五、测试规范

### 5.1 单元测试

```typescript
describe('LeadStateMachine', () => {
  it('should allow transition from new to assigned', () => {
    const result = stateMachine.canTransition('lead', 'new', 'assigned', adminUser);
    expect(result).toBe(true);
  });

  it('should not allow transition from new to following', () => {
    const result = stateMachine.canTransition('lead', 'new', 'following', adminUser);
    expect(result).toBe(false);
  });

  it('should not allow transition from converted (final state)', () => {
    const result = stateMachine.canTransition('lead', 'converted', 'new', adminUser);
    expect(result).toBe(false);
  });

  it('should require reason for invalid transition', async () => {
    await expect(
      stateMachine.transition('lead', 1, 'invalid', salesUser)
    ).rejects.toThrow('Reason is required for invalid transition');
  });
});
```
