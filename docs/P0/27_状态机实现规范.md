## 2. 总规则
- 状态机 ID 规则：`SM-{ENTITY}`。
- 任何状态值必须同时出现在 PRD、DBD、API、页面筛选项中。
- 禁止 `waiting` 与 `pending` 混用；P0 统一按本规范。

## 3. 状态机定义（P0）

### 3.1 SM-customer（客户）
- 枚举：`potential, active, silent, lost`
- 允许流转：
  - potential -> active
  - active -> silent
  - silent -> active
  - active -> lost
  - silent -> lost

### 3.2 SM-lead（线索）
- 枚举：`new, assigned, following, converted, invalid`
- 允许流转：
  - new -> assigned
  - assigned -> following
  - following -> converted
  - new/assigned/following -> invalid

### 3.3 SM-opportunity（商机）
- 阶段枚举：`discovery, qualification, proposal, negotiation`
- 结果枚举：`won, lost`
- 允许流转：
  - discovery -> qualification -> proposal -> negotiation
  - negotiation -> won/lost
  - 禁止跳级推进

### 3.4 SM-conversation（会话）
- 枚举：`queued, active, closed`
- 允许流转：
  - queued -> active
  - active -> closed
  - queued -> closed（超时自动关闭）

### 3.5 SM-ticket（工单）
- 枚举：`pending, assigned, processing, resolved, closed`
- 允许流转：
  - pending -> assigned
  - assigned -> processing
  - processing -> resolved
  - resolved -> closed

### 3.6 SM-task（任务）
- 枚举：`pending, in_progress, completed, cancelled`
- 允许流转：
  - pending -> in_progress
  - in_progress -> completed
  - pending/in_progress -> cancelled

### 3.7 SM-ai_task（AI任务）
- 枚举：`pending, running, succeeded, failed, cancelled`
- 允许流转：
  - pending -> running
  - running -> succeeded
  - running -> failed
  - pending/running -> cancelled

## 5. 前端展示约束
- 标签文本、筛选值与枚举值一一对应。
- 不允许前端自造状态文案映射导致双口径。

## 6. 测试约束
- 每个状态机至少覆盖：
  - 一条合法流转
  - 一条非法流转拦截
- 对非法流转统一返回业务错误码（见 API 错误码文档）。
