# MOY 开发规范文档

## 1. 文档目的
统一 P0 开发阶段的编码、接口、数据、测试与发布规范。

## 2. 强制基线
- 需求基线：`06_PRD`
- 映射基线：`07_RTM`
- 状态机基线：`27_状态机实现规范`
- 接口基线：`11_API`
- 页面基线：`32_页面实现规范`

## 3. 强制规则
- 新功能必须先有 REQ，再有 API/TABLE/TEST/ACPT/TASK 映射。
- 禁止提交未映射到 RTM 的代码。
- 禁止实现 P1/P2 范围功能。
