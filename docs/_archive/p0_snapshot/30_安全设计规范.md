## 2. 安全目标
- 身份可信：JWT 鉴权 + 会话续期。
- 权限可控：RBAC + 数据范围。
- 租户隔离：org_id 全链路强制。
- 数据可追溯：关键操作审计留痕。

## 3. 核心控制
| 控制项 | 实现要求 |
| --- | --- |
| 鉴权 | API/WS 均校验 JWT |
| 授权 | Controller + Service 双层校验 |
| 租户隔离 | Repository 默认 org_id 过滤 |
| 输入校验 | DTO 强校验 + 参数白名单 |
| 输出脱敏 | 敏感字段掩码（密钥、证件、联系方式） |
| 传输安全 | HTTPS/WSS 强制 |
| 审计 | 写操作 100% 审计 |

## 4. 高风险操作保护
- 商机结果标记
- 会话转接/关闭
- 工单关闭
- 角色权限更新
- 系统配置更新

高风险操作要求：权限校验 + 状态机校验 + 审计日志。

## 5. 安全测试重点
- 越权访问拦截（页面与 API）。
- 跨租户数据隔离。
- 非法状态流转拦截。
- AI 脱敏与降级。
- WS 订阅权限校验。
