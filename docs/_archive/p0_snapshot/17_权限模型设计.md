## 2. 权限模型
- 模型：RBAC + 数据范围（self/team/org）。
- 权限 ID 规则：`PERM-{MOD}-{ACTION}`。
- 禁止聚合权限 `conversation:handle`。

## 7. 数据范围规则
- `self`：仅本人负责或本人创建数据。
- `team`：本人 + 团队成员数据。
- `org`：组织内全部数据。
- 组织、角色、权限、系统配置默认 `org` 级。

## 8. 前后端一致性要求
- 后端强校验；前端仅做 UX 层守卫。
- 前端按钮显隐必须与 API 权限映射一致。
- 禁止前端通过隐藏按钮替代后端权限校验。
