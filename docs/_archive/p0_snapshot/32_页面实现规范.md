## 2. 页面实现硬约束
- 页面编号：`PAGE-{MOD}-{NNN}`。
- 路由参数：统一 `{id}`。
- `/` 仅允许重定向到 `/dashboard`。
- 会话页面固定：
  - `PAGE-CNV-001 /conversations`
  - `PAGE-CNV-002 /conversations/{id}`
- 不允许新增 P1/P2 页面进入 P0 路由表。

## 3. 通用交互契约（所有页面）

### 3.1 四态与无权限态
- 加载态：`Skeleton + Spin`。
- 空态：`Empty`（区分“无数据”与“无搜索结果”）。
- 错误态：`Result(status='error') + 重试按钮`。
- 无权限态：`Result(status='403')`，不渲染高权限动作按钮。

### 3.2 列表页统一规格
- 顶部：筛选区 + 搜索框 + 主操作按钮。
- 中部：`Table`（固定列宽、可排序列、状态 Tag 统一组件）。
- 底部：`Pagination`（`page/page_size/total` 对齐 API `meta`）。

### 3.3 表单提交统一规则
- 提交前本地校验 + 服务端错误映射。
- 成功反馈：`message.success("操作成功")`。
- 失败反馈：`message.error(错误文案)`；字段错误回填到 FormItem。
- 二次确认：状态终态动作、删除/关闭类动作必须 `Modal.confirm`。

### 3.4 WebSocket 刷新规则
- 只消费事件，不直接写业务终态。
- 到达事件后触发精准刷新：
  - 列表页更新对应行
  - 详情页增量更新局部 state
- 事件定义以 `29_WebSocket协议规范.md` 为准。

## 5. 动作级权限与按钮显隐总规则
- 页面可见性先校验页面必要权限。
- 按钮显隐再校验动作权限：
  - 会话消息发送：`PERM-CNV-SEND`
  - 会话接入：`PERM-CNV-ACCEPT`
  - 会话转接：`PERM-CNV-TRANSFER`
  - 会话关闭：`PERM-CNV-CLOSE`
- 状态机驱动显隐：
  - 商机 `result` 按钮仅在 `stage=negotiation` 且 `result=null` 可见。
  - 工单 `开始处理` 仅在 `status=assigned` 可见。
  - 工单 `解决` 仅在 `status=processing` 可见。
