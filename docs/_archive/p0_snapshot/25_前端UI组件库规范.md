# MOY 前端 UI 组件库规范

## 1. 文档元信息
| 属性 | 内容 |
| --- | --- |
| 文档编号 | MOY_UI_001 |
| 文档版本 | v2.0 |
| 文档状态 | 已确认（冻结） |
| 日期 | 2026-04-05 |

## 2. 目标
- 为 P0 页面提供统一组件实现口径。
- 保证状态颜色、文案、交互与状态机一致。

## 3. 组件分层
- 基础组件：Button/Input/Select/Table/Modal/Tag
- 业务组件：
  - CustomerStatusTag
  - LeadStatusTag
  - OpportunityStageStepper
  - ConversationActionBar
  - TicketStatusPanel
  - PermissionGuard

## 4. 状态展示口径（必须一致）
| 实体 | 枚举 |
| --- | --- |
| customer | potential, active, silent, lost |
| lead | new, assigned, following, converted, invalid |
| opportunity | discovery, qualification, proposal, negotiation, won, lost |
| conversation | queued, active, closed |
| ticket | pending, assigned, processing, resolved, closed |
| task | pending, in_progress, completed, cancelled |
| ai_task | pending, running, succeeded, failed, cancelled |

## 5. 页面级组件绑定
| PAGE ID | 关键组件 |
| --- | --- |
| PAGE-CM-001 | CustomerTable, CustomerFilter |
| PAGE-LM-001 | LeadTable, LeadAssignPanel |
| PAGE-OM-002 | OpportunityStageStepper |
| PAGE-CNV-002 | ConversationMessageList, ConversationActionBar, SmartReplyPanel |
| PAGE-TK-002 | TicketStatusPanel, TicketActionPanel |

## 6. 禁止事项
- 禁止组件内硬编码与状态机不一致的状态值。
- 禁止出现 `waiting`、`win`、`lose` 等非基线值。
- 禁止在页面层跳过权限守卫直接渲染高权限动作按钮。
