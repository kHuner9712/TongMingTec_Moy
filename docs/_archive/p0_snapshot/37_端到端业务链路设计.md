## 2. P0 链路总览
- 获客：线索录入 -> 分配 -> 跟进 -> 转化商机
- 谈单：商机创建 -> 阶段推进 -> 结果标记（won/lost）
- 服务：会话接入 -> 发送消息 -> 转接/关闭 -> 转工单
- 售后：工单创建 -> 分配 -> 处理 -> 解决 -> 关闭
- 智能：会话触发 AI 回复 -> 人工确认发送
- 系统：权限配置 -> 系统配置 -> 审计追踪

## 3. E2E-01 线索 -> 商机 -> 结果

### 3.1 链路步骤
1. `API-LM-002` 创建线索（`TABLE-LM-001 status=new`）。
2. `API-LM-003` 分配线索（`status=assigned`）。
3. `API-LM-004` 添加跟进（`TABLE-LM-002` 新增记录）。
4. `API-LM-005` 转化商机（`TABLE-OM-001` 新增，线索 `status=converted`）。
5. `API-OM-005` 推进到 `negotiation`。
6. `API-OM-006` 标记 `won/lost`。

### 3.2 关键断言
- 商机 `result` 仅 `won/lost`。
- 线索转化使用单事务，禁止“商机已创建但线索未转化”。
- 全链路写审计。

### 3.3 页面对应
- `PAGE-LM-001`、`PAGE-LM-002`、`PAGE-OM-001`、`PAGE-OM-002`。

### 3.4 用例对应
- TEST: `TEST-LM-001~004`、`TEST-OM-001~003`
- ACPT: `ACPT-LM-001~004`、`ACPT-OM-001~003`

## 4. E2E-02 会话 -> 工单

### 4.1 链路步骤
1. `API-CNV-001/002/003` 查看会话与历史消息。
2. `API-CNV-005` 接入会话（queued -> active）。
3. `API-CNV-004` 发送消息（写入 `TABLE-CNV-002`）。
4. `API-CNV-006` 转接（可选）。
5. `API-CNV-008` 从会话创建工单。
6. `API-TK-004/005/006/007` 工单闭环。

### 4.2 关键断言
- 会话权限必须细分 SEND/ACCEPT/TRANSFER/CLOSE。
- 会话转工单后 ticket.source_type/source_id 正确。
- WS 事件与页面状态一致。

### 4.3 页面对应
- `PAGE-CNV-001`、`PAGE-CNV-002`、`PAGE-TK-001`、`PAGE-TK-002`。

### 4.4 用例对应
- TEST: `TEST-CNV-001~005`、`TEST-TK-001~004`
- ACPT: `ACPT-CNV-001~005`、`ACPT-TK-001~004`

## 5. E2E-03 AI 智能回复

### 5.1 链路步骤
1. 操作员在 `PAGE-CNV-002` 点击“智能回复”（`API-AI-001`）。
2. 创建 `ai_task`（pending -> running）。
3. 任务完成（succeeded/failed），WS 推送 `ai.task.status.changed`。
4. succeeded 时草稿回填，操作员确认后发送（`API-CNV-004`）。

### 5.2 关键断言
- AI 输出不自动发送，不自动写业务终态。
- 失败时可人工接管。
- 成本与 token 字段落库完整。

### 5.3 用例对应
- TEST: `TEST-AI-001/002`
- ACPT: `ACPT-AI-001/002`

## 6. E2E-04 系统配置与权限配置

### 6.1 链路步骤
1. `PAGE-USR-002` 加载角色与权限（API-USR-004/005）。
2. 更新角色权限（API-USR-002）。
3. `PAGE-SYS-002` 更新系统配置（API-SYS-003）。
4. `PAGE-AUD-001` 检索审计（API-AUD-001）确认变更记录。

### 6.2 关键断言
- 权限更新后按钮显隐即时生效。
- 配置更新发生并发冲突时返回 `CONFLICT_VERSION`。
- 审计记录包含 before/after。

### 6.3 用例对应
- TEST: `TEST-USR-002`、`TEST-SYS-002`、`TEST-AUD-001`
- ACPT: `ACPT-USR-002`、`ACPT-SYS-002`、`ACPT-AUD-001`

## 7. 全链路非功能断言
- 每步请求可通过 `request_id` 串联。
- 跨租户访问均被阻断。
- 状态机非法流转全部返回对应错误码。
- 核心链路步骤 P95 延迟在 NFR 阈值内。

## 8. P1/P2 出界声明
- 合同/订单/发票/续费、知识库、看板、自动化规则均不在本链路中。
