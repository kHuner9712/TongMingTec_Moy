## 2. 测试范围
- 仅覆盖 P0（35 条 REQ 对应 35 条 TEST 与 35 条 ACPT）。
- 不纳入 P1/P2 用例。

## 3. 测试分层
| 分层 | 目标 | 产出 |
| --- | --- | --- |
| API 合约测试 | 校验 schema、状态码、错误码、幂等 | `api/*.spec.ts` |
| 状态机测试 | 校验合法/非法流转 | `state-machine/*.spec.ts` |
| 权限与租户测试 | 校验 PERM 与 org_id 隔离 | `authz/*.spec.ts` |
| WebSocket 联动测试 | 校验事件与前端刷新 | `ws/*.spec.ts` |
| E2E 测试 | 校验跨模块链路闭环 | `e2e/*.spec.ts` |
| ACPT 业务验收 | 校验业务可交付 | 验收记录单 |

## 4. 必达覆盖要求
- REQ -> TEST 覆盖率 100%。
- REQ -> ACPT 覆盖率 100%。
- 每个状态机至少覆盖 1 条合法 + 1 条非法流转。
- 每个写接口至少 1 条审计断言。
- 每个核心链路至少 1 条跨租户阻断用例。

## 5. 核心 E2E 链路（强制）
1. 线索 -> 商机（REQ-LM-001~004 + REQ-OM-001~003）
2. 会话 -> 工单（REQ-CNV-001~005 + REQ-TK-001~004）
3. AI 智能回复（REQ-AI-001~002）
4. 系统配置（REQ-SYS-002）
5. 权限配置（REQ-USR-002 + REQ-ORG-001）

## 6. 执行环境
| 环境 | 说明 |
| --- | --- |
| DEV | 开发自测，允许 mock 外部 AI |
| TEST | 集成测试，连接真实 PostgreSQL/Redis/WebSocket |
| PROD-STAGING | 发布前回归与 ACPT |

## 7. 发布门禁（测试维度）
- 所有 P0 TEST 用例通过。
- 所有 P0 ACPT 用例通过并签字。
- 关键链路无 P0 阻断级缺陷。
- 错误码、状态机、权限、租户隔离检查全部通过。
