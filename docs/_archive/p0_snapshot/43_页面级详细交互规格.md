## 2. 通用实现规则（适用于所有 PAGE）
- 表格分页参数与 API `meta` 字段保持一致。
- 筛选条件变化后重置到 `page=1`。
- 所有写操作成功后刷新当前页数据；失败保留用户输入。
- 字段级错误使用 `Form.Item help + validateStatus` 渲染。
- 错误码展示策略来自 `28_API错误码清单.md`。

## 13. PAGE-CNV-002 会话详情 `/conversations/{id}`

### 13.1 页面目标
- 处理会话消息、接入转接关闭、会话转工单、AI 智能回复。

### 13.2 页面区块
| 区块 | 说明 | API |
| --- | --- | --- |
| 头部信息区 | 状态、渠道、客户、处理人 | API-CNV-002 |
| 消息流区 | 历史与实时消息 | API-CNV-003/004 |
| 操作栏 | 接入/转接/关闭/转工单 | API-CNV-005/006/007/008 |
| AI 面板 | 触发智能回复、查看任务状态 | API-AI-001/002 |

### 13.3 消息发送表单
| 字段 | 必填 | 规则 |
| --- | --- | --- |
| content_type | 是 | text/image/file/template |
| content | 是 | text 时 1~4000 |
| attachments | 否 | array |
| client_msg_id | 否 | <=64（幂等） |
| is_internal | 否 | boolean |

### 13.4 动作按钮矩阵
| 按钮 | 权限 | API | 显隐条件 | 二次确认 |
| --- | --- | --- | --- | --- |
| 发送消息 | PERM-CNV-SEND | API-CNV-004 | status=active | 否 |
| 接入会话 | PERM-CNV-ACCEPT | API-CNV-005 | status=queued | 否 |
| 转接会话 | PERM-CNV-TRANSFER | API-CNV-006 | status=active | 是 |
| 关闭会话 | PERM-CNV-CLOSE | API-CNV-007 | status in queued/active | 是 |
| 创建工单 | PERM-CNV-CREATE_TICKET | API-CNV-008 | status in active/closed | 是 |
| 智能回复 | PERM-AI-EXECUTE | API-AI-001 | status=active | 否 |

### 13.5 弹窗/抽屉规则
- 转接弹窗字段：`target_user_id* reason`。
- 关闭弹窗字段：`close_reason*`。
- 创建工单弹窗字段：`title* description priority*`。

### 13.6 成败反馈文案模式
- 成功：`{动作名}成功`。
- 失败：
  - 权限不足：`无操作权限`。
  - 状态非法：`当前会话状态不允许该操作`。
  - AI 超时：`AI 生成超时，请手动回复`。

### 13.7 WebSocket 更新策略
| 事件 | 前端动作 |
| --- | --- |
| conversation.message.created | append 消息；如 seq 重复则丢弃 |
| conversation.status.changed | 更新 header 状态 + 操作按钮显隐 |
| ticket.created | 展示“工单已创建”并显示 ticket 链接 |
| ai.task.status.changed | 更新 AI 面板状态、填充回复草稿 |

### 13.8 Store 策略
- 会话消息按 `seq_no` 升序存储。
- 已存在 `message.id` 或 `client_msg_id` 则去重。

### 13.9 组件绑定
- `ConversationMessageList`、`ConversationActionBar`、`SmartReplyPanel`、`PermissionGuard`。
