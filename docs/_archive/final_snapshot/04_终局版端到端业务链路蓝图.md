## 一、文档目的

本文档作为 MOY 项目的**终局版端到端业务链路蓝图**，用于：

1. 定义 MOY 终局版完整的端到端业务链路
2. 明确七大业务链路的完整流程与关键节点
3. 定义链路间的数据流转与触发关系
4. 建立业务链路与业务域的映射关系
5. 指导 P0/P1 阶段的链路实现与扩展规划

**核心原则：** 每个链路都必须形成闭环，链路之间无缝衔接，实现客户经营全生命周期覆盖。

## 三、术语定义

### 3.1 核心术语

| 术语 | 定义 |
|------|------|
| 业务链路 | 从业务起点到终点的完整流程路径，包含多个业务节点 |
| 业务节点 | 业务链路中的关键环节，具有明确的输入输出和状态变更 |
| 链路闭环 | 业务链路从起点到终点的完整循环，确保业务目标达成 |
| 链路触发 | 一个链路节点完成后触发下一个链路节点或另一条链路 |
| 数据流转 | 业务数据在链路节点之间的传递与转换 |
| 状态机 | 实体状态流转的规则定义，控制业务节点间的转换 |

### 3.2 链路术语

| 术语 | 定义 |
|------|------|
| 获客链路 | 从线索获取到线索转化的完整流程 |
| 沟通链路 | 从客户触达到会话结束的完整流程 |
| 谈单链路 | 从商机创建到商机成交/失败的完整流程 |
| 成交链路 | 从商机赢单到订单完成的完整流程 |
| 售后链路 | 从工单创建到工单关闭的完整流程 |
| 续费链路 | 从续费提醒到续费完成的完整流程 |
| 增购链路 | 从增购识别到增购成交的完整流程 |
| 转介绍链路 | 从转介绍记录到新客户成交的完整流程 |

### 3.3 实体术语

| 术语 | 定义 |
|------|------|
| 线索(Lead) | 潜在客户信息，尚未确认购买意向 |
| 客户(Customer) | 已确认身份的客户实体，可能有多笔交易 |
| 联系人(Contact) | 客户的具体联系人，一个客户可有多个联系人 |
| 商机(Opportunity) | 具有明确购买意向的销售机会 |
| 会话(Conversation) | 客户与企业之间的沟通会话 |
| 工单(Ticket) | 客户问题或服务请求的处理记录 |
| 合同(Contract) | 客户与企业签署的法律协议 |
| 订单(Order) | 客户购买产品/服务的交易记录 |
| 回款(Payment) | 客户支付款项的记录 |
| 发票(Invoice) | 开具给客户的发票记录 |

| **链路终点** | 线索转化/线索失效 |
