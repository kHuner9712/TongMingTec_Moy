## 一、文档目的

本文档作为 MOY 项目的**终局版业务域与能力蓝图**，用于：

1. 定义 MOY 终局版完整的业务域划分
2. 明确各业务域的核心能力清单
3. 建立能力分级体系（平台能力/行业能力/客户自定义能力）
4. 定义业务域之间的关联关系与协作机制
5. 指导 P0/P1 阶段的能力规划与架构设计

**核心原则：** 业务域划分遵循客户经营全链路，能力设计遵循分层解耦原则。

## 三、术语定义

### 3.1 核心术语

| 术语 | 定义 |
|------|------|
| 业务域 | 业务能力的逻辑分组，代表一个完整的业务领域 |
| 能力 | 业务域内可独立交付价值的最小功能单元 |
| 能力项 | 能力的具体实现项，可配置、可扩展 |
| 平台能力 | 所有客户通用的基础能力，由平台统一提供 |
| 行业能力 | 特定行业所需的差异化能力，按行业包提供 |
| 客户自定义能力 | 客户根据自身需求定制的能力，支持二次开发 |

### 3.2 能力分级术语

| 术语 | 定义 | 特征 |
|------|------|------|
| L1-平台核心能力 | 平台必须提供的基础能力 | 不可关闭、不可定制、全客户可用 |
| L2-平台扩展能力 | 平台可选提供的增强能力 | 可开关、可配置、按需启用 |
| L3-行业能力 | 特定行业专属能力 | 按行业包授权、行业定制化 |
| L4-客户自定义能力 | 客户自行开发的能力 | 支持二次开发、API扩展 |

### 3.3 业务域术语

| 术语 | 定义 |
|------|------|
| 获客域 | 负责线索获取、渠道管理、营销活动的业务域 |
| 沟通域 | 负责会话管理、消息处理、智能回复的业务域 |
| 谈单域 | 负责商机管理、报价管理、合同管理的业务域 |
| 成交域 | 负责订单管理、回款管理、发票管理的业务域 |
| 售后域 | 负责工单管理、服务记录、客户满意度的业务域 |
| 续费域 | 负责续费管理、增购管理、客户健康度的业务域 |
| 运营域 | 负责知识库、数据看板、自动化规则的业务域 |

### 4.2 业务域定位矩阵

| 业务域 | 域定位 | 核心价值 | 关键角色 | 业务阶段 |
|--------|--------|----------|----------|----------|
| **获客域** | 流量入口 | 线索获取与转化 | 市场人员、销售经理 | 售前 |
| **沟通域** | 触达枢纽 | 客户互动与响应 | 销售人员、客服人员 | 售前/售后 |
| **谈单域** | 转化引擎 | 商机推进与成交 | 销售人员、销售经理 | 售前 |
| **成交域** | 交易闭环 | 订单履约与回款 | 销售人员、财务人员 | 成交 |
| **售后域** | 服务保障 | 问题解决与满意度 | 客服人员、技术支持 | 售后 |
| **续费域** | 价值延伸 | 客户留存与增值 | 客户成功、销售人员 | 运营 |
| **运营域** | 能力支撑 | 知识、数据、自动化 | 运营人员、管理员 | 全阶段 |

## 六、能力分级体系

### 6.1 能力分级定义

| 分级 | 名称 | 定义 | 特征 | 授权方式 |
|------|------|------|------|----------|
| **L1** | 平台核心能力 | 所有客户必须可用的基础能力 | 不可关闭、不可定制、全客户可用 | 平台默认提供 |
| **L2** | 平台扩展能力 | 平台可选提供的增强能力 | 可开关、可配置、按需启用 | 按套餐授权 |

| **L4** | 客户自定义能力 | 客户自行开发的能力 | 支持二次开发、API扩展 | 开发者授权 |

### 8.1 对 P0 阶段的影响

#### 8.1.1 能力规划影响

| 业务域 | P0 必须实现 | P0 可选实现 | P0 暂不实现 |
|--------|-------------|-------------|-------------|
| 获客域 | 线索录入、导入、分配、转化 | 线索清洗、回收、去重 | 线索评分、渠道归因、活动优化 |
| 沟通域 | 会话接入、分配、转接、结束、消息收发、自动回复、话术库 | 会话监控、消息搜索、AI回复、话术推荐 | 会话质检、情绪分析、多轮对话、行业话术 |
| 谈单域 | 商机创建、阶段、跟进 | 商机分析、暂停、输单 | 商机预测、报价分析、预测模型 |
| 成交域 | 订单创建、查询 | 订单审批、变更、取消、统计 | 回款计划、发票管理、电子签署 |
| 售后域 | 工单创建、分配、流转、记录创建、满意度调查 | 工单升级、转派、统计、SLA配置 | 工单模板、满意度改进 |
| 续费域 | - | - | 全部能力 |
| 运营域 | 知识录入、分类、检索、核心指标 | 知识问答、推荐、统计、趋势分析 | 知识图谱、自定义报表、自动化规则、AI工作台 |

#### 8.1.2 架构设计影响

| 影响维度 | 具体影响 | 应对策略 |
|----------|----------|----------|
| 业务域边界 | P0 需明确各业务域边界，避免耦合 | 采用领域驱动设计，明确聚合根与边界上下文 |
| 能力扩展性 | P0 能力需支持后续扩展 | 采用插件化架构，预留扩展点 |
| 数据模型 | 数据模型需支持跨域关联 | 采用统一客户ID，建立域间关联关系 |
| API设计 | API 需支持能力分级 | 采用API版本控制，支持能力开关 |

#### 8.1.3 技术债务规避

| 风险项 | 规避策略 |
|--------|----------|
| 业务域耦合 | 采用领域驱动设计，明确边界上下文 |
| 能力不可扩展 | 采用插件化架构，预留扩展接口 |
| 数据孤岛 | 采用统一客户ID，建立数据关联 |
| API不兼容 | 采用版本控制，保持向后兼容 |

### 8.2 对 P1 阶段的影响

#### 8.2.1 能力规划影响

| 业务域 | P1 必须实现 | P1 可选实现 | P1 暂不实现 |
|--------|-------------|-------------|-------------|
| 获客域 | 线索评分、渠道追踪 | 活动创建、触达、追踪、表单设计、发布 | 渠道归因、渠道成本、活动优化、表单分析 |
| 沟通域 | 会话质检、AI回复增强、话术推荐增强 | 情绪分析、多轮对话、行业话术 | - |
| 谈单域 | 报价创建、审批、版本、合同创建、审批、归档 | 商机预测、商机协作、报价模板、合同模板 | 报价分析、销售预测 |
| 成交域 | 回款计划、提醒、核销、发票申请、审批、记录 | 回款分析、发票统计 | 电子签署、签署流程 |
| 售后域 | SLA监控、预警、报告、满意度统计 | 工单模板、满意度预警 | 满意度改进 |
| 续费域 | 续费提醒、跟进、健康度评估、预警 | 续费预测、增购识别、推荐、转介绍记录 | 增购分析、转介绍激励、流失预警 |
| 运营域 | 知识问答增强、自定义报表、数据预警、规则配置、执行 | 行业知识、BI集成、规则模板 | 知识图谱、AI工作台 |

#### 8.2.2 功能依赖关系

| P1 功能 | P0 依赖 | 依赖说明 |
|---------|---------|----------|
| 线索评分 | 线索管理 | 需要线索数据支撑评分模型 |
| 会话质检 | 会话管理 | 需要会话数据支撑质检模型 |
| 报价管理 | 商机管理 | 需要商机数据关联报价 |
| 合同管理 | 商机管理 | 需要商机数据关联合同 |
| 回款管理 | 订单管理 | 需要订单数据关联回款 |
| 续费管理 | 客户管理、订单管理 | 需要客户和订单数据支撑 |
| 健康度评估 | 客户管理、工单管理 | 需要客户行为和服务数据 |
| 自动化规则 | 全域数据 | 需要各业务域数据触发规则 |

#### 8.2.3 数据依赖关系

| P1 数据需求 | P0 数据准备 |
|-------------|-------------|
| 线索评分数据 | 线索基础信息、跟进记录 |
| 会话质检数据 | 会话记录、消息内容 |
| 报价关联数据 | 商机数据、产品数据 |
| 合同关联数据 | 商机数据、客户数据 |
| 回款关联数据 | 订单数据、客户数据 |
| 健康度评估数据 | 客户行为、服务记录、使用数据 |
