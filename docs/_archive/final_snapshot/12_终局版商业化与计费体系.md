## 三、术语定义

### 3.1 核心术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 套餐 | Plan | 产品功能的组合包，对应不同的价格和服务等级 |
| 席位 | Seat | 系统用户账号，计费的基本单位 |
| 计费周期 | Billing Cycle | 计费的时间单位，如月付、年付 |
| 账单 | Bill | 租户应付费用明细，包含各项计费项 |
| 订阅 | Subscription | 租户购买的套餐和服务 |
| 续费 | Renewal | 延长订阅有效期的行为 |
| 升级 | Upgrade | 提升套餐等级或增加服务 |
| 降级 | Downgrade | 降低套餐等级或减少服务 |
| 增购 | Add-on | 在现有套餐基础上增加额外服务 |
| 欠费 | Overdue | 账户余额不足以支付应付费用 |
| 停服 | Service Suspension | 因欠费等原因暂停服务 |

### 3.2 计费术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 按席位计费 | Per-Seat Billing | 按用户数量计费的模式 |
| 按量计费 | Usage-Based Billing | 按实际使用量计费的模式 |
| 按模块计费 | Module Billing | 按功能模块单独计费的模式 |
| 按 Agent 计费 | Agent Billing | 按 AI Agent 调用量计费的模式 |
| 按调用量计费 | API Call Billing | 按 API 调用次数计费的模式 |
| 阶梯定价 | Tiered Pricing | 根据用量区间采用不同单价的定价方式 |
| 包年优惠 | Annual Discount | 年付相比月付的折扣优惠 |

### 3.3 租户状态术语

| 术语 | 英文 | 定义 |
|------|------|------|
| 试用中 | Trial | 租户处于免费试用期 |
| 正常 | Active | 租户已付费，服务正常 |
| 欠费 | Overdue | 租户欠费，服务受限 |
| 停服 | Suspended | 租户服务暂停 |
| 过期 | Expired | 租户订阅已到期 |
| 注销 | Cancelled | 租户已注销 |

## 八、升级/降级规则

### 8.1 升级规则

#### 8.1.1 升级场景

| 场景 | 说明 |
|------|------|
| 套餐升级 | 从低级套餐升级到高级套餐 |
| 席位增加 | 增加购买的席位数量 |
| 模块增购 | 增购可选功能模块 |
| 周期延长 | 从月付升级为年付 |

#### 8.1.2 升级费用计算

```
升级费用计算公式：

升级费用 = 新套餐价格 × 剩余天数 / 周期天数 - 原套餐剩余价值

示例：
- 当前套餐：标准版 ¥299/月，已使用 15 天
- 升级套餐：专业版 ¥599/月
- 剩余天数：15 天

升级费用 = ¥599 × 15/30 - ¥299 × 15/30
         = ¥299.5 - ¥149.5
         = ¥150
```

#### 8.1.3 升级生效时间

| 升级类型 | 生效时间 |
|----------|----------|
| 套餐升级 | 立即生效 |
| 席位增加 | 立即生效 |
| 模块增购 | 立即生效 |
| 周期延长 | 下个周期生效 |

#### 8.1.4 升级限制

| 限制项 | 说明 |
|--------|------|
| 套餐兼容 | 只能升级到更高级的套餐 |
| 席位下限 | 升级后席位不能低于新套餐最低要求 |
| 数据保留 | 升级不影响已有数据 |

### 8.2 降级规则

#### 8.2.1 降级场景

| 场景 | 说明 |

#### 8.2.2 降级费用处理

```
降级费用处理：

1. 不退款原则：降级不退还差价
2. 费用抵扣：剩余费用转为账户余额，可用于后续消费
3. 下周期生效：降级在当前计费周期结束后生效

示例：
- 当前套餐：专业版 ¥599/月，已使用 15 天
- 降级套餐：标准版 ¥299/月
- 剩余天数：15 天

处理方式：
- 当前周期继续使用专业版
- 下个周期自动切换为标准版
- 不退还差价
```
