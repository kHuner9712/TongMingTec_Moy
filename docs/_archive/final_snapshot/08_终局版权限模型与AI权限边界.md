## 一、文档目标

本文档定义 MOY 终局版权限模型与 AI 权限边界，作为企业级 AI 原生客户管理系统的安全基线，用于：

1. 定义 RBAC 权限模型的核心架构与规则
2. 明确数据权限范围与字段权限粒度
3. 划定 AI 系统的操作边界与限制规则
4. 规范敏感操作的审计要求
5. 建立权限配置化策略框架
6. 确保 AI 与人类协作的安全可控

| 规则编号 | 规则名称 | 规则描述 | 适用场景 |
|----------|----------|----------|----------|

#### 4.2.3 继承实现

```typescript
interface RoleInheritanceConfig {
  roleId: bigint;
  parentRoleId: bigint;
  inheritType: 'full' | 'partial' | 'none';
  excludedPermissions: string[];
  additionalPermissions: string[];
}

async function resolveInheritedPermissions(roleId: bigint): Promise<Set<string>> {
  const permissions = new Set<string>();
  const visited = new Set<bigint>();
  
  async function collectPermissions(currentRoleId: bigint) {
    if (visited.has(currentRoleId)) return;
    visited.add(currentRoleId);
    
    const role = await getRole(currentRoleId);
    
    const directPermissions = await getRolePermissions(currentRoleId);
    directPermissions.forEach(p => permissions.add(p));
    
    if (role.parentRoleId) {
      await collectPermissions(role.parentRoleId);
    }
  }
  
  await collectPermissions(roleId);
  return permissions;
}
```

### 4.3 权限冲突解决规则

#### 4.3.1 冲突场景定义

| 冲突类型 | 冲突描述 | 示例 |
|----------|----------|------|
| 多角色冲突 | 用户拥有多个角色，权限不一致 | 角色A有删除权限，角色B无删除权限 |
| 继承冲突 | 子角色与父角色权限冲突 | 父角色禁止删除，子角色允许删除 |
| 数据范围冲突 | 不同角色的数据范围不一致 | 角色A为全部数据，角色B为仅本人 |
| 字段权限冲突 | 不同角色对同一字段权限不同 | 角色A可编辑，角色B只读 |
| 时间限制冲突 | 临时权限与永久权限冲突 | 临时授权已过期 |

#### 4.3.2 冲突解决策略

| 策略编号 | 策略名称 | 策略描述 | 优先级 |
|----------|----------|----------|--------|
| RES-001 | 就高原则 | 权限冲突时，取权限更大者 | 1 |
| RES-002 | 就宽原则 | 数据范围冲突时，取范围更宽者 | 2 |
| RES-003 | 否决优先 | 存在禁止权限时，禁止优先 | 3 |
| RES-004 | 时间优先 | 临时权限优先于永久权限判断 | 4 |
| RES-005 | 显式优先 | 显式授权优先于隐式继承 | 5 |

#### 4.3.3 冲突解决实现

```typescript
interface PermissionConflict {
  userId: bigint;
  resourceType: string;
  action: string;
  conflictingRoles: bigint[];
  conflictType: 'allow_deny' | 'scope_mismatch' | 'field_mismatch';
}

async function resolvePermissionConflict(conflict: PermissionConflict): Promise<PermissionDecision> {
  const decisions = await Promise.all(
    conflict.conflictingRoles.map(roleId => 
      checkPermission(roleId, conflict.resourceType, conflict.action)
    )
  );
  
  const hasDeny = decisions.some(d => d === 'deny');
  if (hasDeny) {
    return { decision: 'deny', reason: 'RES-003: 否决优先' };
  }
  
  const hasAllow = decisions.some(d => d === 'allow');
  if (hasAllow) {
    return { decision: 'allow', reason: 'RES-001: 就高原则' };
  }
  
  return { decision: 'deny', reason: '默认拒绝' };
}

async function resolveDataScopeConflict(userId: bigint, resourceType: string): Promise<DataScope> {
  const roles = await getUserRoles(userId);
  const scopes = await Promise.all(
    roles.map(role => getDataScope(role.id, resourceType))
  );
  
  const scopePriority = {
    'all': 5,
    'department_and_sub': 4,
    'department': 3,
    'custom': 2,
    'self': 1
  };
  
  let maxScope: DataScope = { type: 'self', value: null };
  
  for (const scope of scopes) {
    if (scopePriority[scope.type] > scopePriority[maxScope.type]) {
      maxScope = scope;
    }
  }
  
  return maxScope;
}
```

### 5.3 数据范围实现

#### 5.3.1 数据范围拦截器

```typescript
@Injectable()
export class DataScopeInterceptor implements NestInterceptor {
  constructor(private readonly dataScopeService: DataScopeService) {}
  
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const request = context.switchToHttp().getRequest();
    const user = request.user;
    const resourceType = this.getResourceType(context);
    
    return from(this.applyDataScope(user, resourceType)).pipe(
      switchMap(scope => {
        request.dataScope = scope;
        return next.handle();
      })
    );
  }
  
  private async applyDataScope(user: User, resourceType: string): Promise<DataScopeFilter> {
    const scope = await this.dataScopeService.getUserDataScope(user.id, resourceType);
    
    switch (scope.type) {
      case 'all':
        return { where: 'org_id = :orgId', params: { orgId: user.orgId } };
        
      case 'department_and_sub':
        const subDeptIds = await this.getSubDepartmentIds(user.departmentId);
        return { 
          where: 'department_id IN (:deptIds)', 
          params: { deptIds: [user.departmentId, ...subDeptIds] } 
        };
        
      case 'department':
        return { 
          where: 'department_id = :deptId', 
          params: { deptId: user.departmentId } 
        };
        
      case 'self':
        return { 
          where: '(owner_id = :userId OR created_by = :userId)', 
          params: { userId: user.id } 
        };
        
      case 'custom':
        return this.buildCustomScope(scope);
        
      default:
        return { where: '1 = 0', params: {} };
    }
  }
}
```

### 6.4 字段权限实现

```typescript
interface FieldPermission {
  fieldCode: string;
  permission: 'visible' | 'editable' | 'readonly' | 'hidden' | 'masked';
  maskRule?: MaskRule;
}

interface MaskRule {
  type: 'phone' | 'email' | 'id_card' | 'bank_card' | 'name' | 'address' | 'custom';
  keepPrefix: number;
  keepSuffix: number;
  maskChar: string;
}

class FieldPermissionService {
  async applyFieldPermissions(
    entity: any,
    entityType: string,
    userId: bigint
  ): Promise<any> {
    const permissions = await this.getFieldPermissions(userId, entityType);
    const result = { ...entity };
    
    for (const [field, value] of Object.entries(entity)) {
      const perm = permissions.find(p => p.fieldCode === field);
      
      if (!perm) continue;
      
      switch (perm.permission) {
        case 'hidden':
          delete result[field];
          break;
        case 'masked':
          result[field] = this.applyMask(value, perm.maskRule);
          break;
        case 'visible':
        case 'readonly':
        case 'editable':
          break;
      }
    }
    
    return result;
  }
  
  private applyMask(value: string, rule: MaskRule): string {
    if (!value) return value;
    
    const len = value.length;
    const prefix = value.substring(0, rule.keepPrefix);
    const suffix = value.substring(len - rule.keepSuffix);
    const maskLen = len - rule.keepPrefix - rule.keepSuffix;
    const mask = rule.maskChar.repeat(Math.max(0, maskLen));
    
    return prefix + mask + suffix;
  }
}
```

class AIApprovalService {
  async evaluateOperation(operation: AIOperation): Promise<ApprovalDecision> {
    const rules = await this.getApplicableRules(operation);

      if (this.matchesConditions(operation, rule.triggerConditions)) {
        return {
          requiresApproval: true,
          rule: rule,
          riskLevel: this.calculateRiskLevel(operation, rule)
        };
      }

      riskLevel: this.calculateDefaultRiskLevel(operation)
    };

class AISensitiveOperationGuard {
  async checkOperation(operation: AIOperation): Promise<OperationCheckResult> {
    const limit = AI_SENSITIVE_OPERATION_LIMITS.find(
      l => l.operation === operation.action
    );
    
    if (!limit) {
      return { allowed: true, reason: '非敏感操作' };
    }
    
    if (limit.prohibited) {
      await this.auditProhibitedAttempt(operation, limit);
      return { 
        allowed: false, 
        reason: limit.prohibitionReason || '操作被禁止' 
      };
    }
    
    if (limit.maxCount && operation.count > limit.maxCount) {
      return { 
        allowed: false, 
        reason: `操作数量超过限制（最大${limit.maxCount}条）` 
      };
    }
    
    if (limit.maxAmount && operation.amount > limit.maxAmount) {
      return { 
        allowed: false, 
        reason: `操作金额超过限制（最大${limit.maxAmount}）` 
      };
    }
    
    if (limit.requiresApproval) {
      const approvalRequest = await this.createApprovalRequest(operation, limit);
      return { 
        allowed: false, 
        requiresApproval: true,
        approvalRequestId: approvalRequest.id,
        reason: '敏感操作需要审批' 
      };
    }
    
    if (limit.auditRequired) {
      await this.auditSensitiveOperation(operation, limit);
    }
    
    return { allowed: true, reason: '敏感操作已记录' };
  }
}
