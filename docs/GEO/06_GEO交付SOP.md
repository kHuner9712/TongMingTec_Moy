# MOY GEO 交付 SOP

## 1. 文档用途

本文档定义 MOY GEO 服务从售前到续费的标准操作流程。每条环节有明确的输入、输出、负责人和质量标准。

执行原则：
- 先诊断后收费
- 每个阶段交付物明确
- 客户全程可感知进展
- 不承诺做不到的结果

## 2. 流程总图

```
阶段 1   客户信息收集        → [01_客户资料收集表] 完成
阶段 2   初始诊断            → [02_诊断报告] 交付客户
阶段 3   方案报价确认        → 签约 / 收款
阶段 4   资料补全与确认      → [03_品牌事实资产包] 客户签字
阶段 5   内容生产            → [04_选题库] + 内容稿件
阶段 6   发布与分发建议      → 内容上线 + 分发报告
阶段 7   月度监测            → [05_月度监测报告]
阶段 8   月度复盘            → 复盘会议 + 下月计划
阶段 9   续费 / 升级         → 续费方案确认
```

## 3. 各阶段详解

### 阶段 1：客户信息收集

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 客户表达诊断意向 |
| 负责人 | 销售 / GEO 顾问 |
| 输入 | [01_客户资料收集表](./01_客户资料收集表.md) |
| 输出 | 客户资料表填写完成 |
| 耗时 | 1-3 个工作日（含客户响应时间） |
| 质量标准 | 核心字段（公司名/品牌/官网/行业/产品/竞品）必须填写 |
| 客户动作 | 填写资料表 |

### 阶段 2：初始诊断

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 客户资料表填写完成 |
| 负责人 | GEO 交付 |
| 输入 | 客户资料表 |
| 输出 | [02_AI可见度诊断报告](./02_AI可见度诊断报告模板.md) PDF |
| 耗时 | 3-5 个工作日 |
| 质量标准 | 覆盖 ≥5 个平台，≥20 个测试问题 |
| 客户动作 | 接收报告，安排诊断解读会 |

**操作步骤**：

1. 基于客户资料表，设计 20-30 个测试问题，覆盖品牌认知、行业推荐、竞品对比、地域服务等维度。
2. 逐一在目标 AI 平台中测试，截图或记录原文。
3. 填写诊断报告模板中的所有表格。
4. 输出综合评分、问题列表和优化建议。
5. 内部评审通过后发送客户。

### 阶段 3：方案报价确认

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 诊断报告交付完成，客户有意向继续 |
| 负责人 | 销售 / GEO 顾问 |
| 输入 | 诊断报告中的优化建议 |
| 输出 | 签约合同 / 服务协议 |
| 耗时 | 1-2 个工作日 |
| 客户动作 | 确认方案、签约、付款 |

**报价模板**：见 [07_GEO销售话术与报价](./07_GEO销售话术与报价.md)

### 阶段 4：资料补全与品牌事实资产建设

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 签约付款完成 |
| 负责人 | GEO 交付 |
| 输入 | 客户资料表 + 诊断报告 + 客户补充材料 |
| 输出 | [03_品牌事实资产包](./03_品牌事实资产包模板.md) 客户签字的 PDF |
| 耗时 | 3-7 个工作日 |
| 质量标准 | 所有章节填写完整，可公开/不可使用材料标记清晰，客户签字确认 |
| 客户动作 | 提供补充材料、审核资产包、签字确认 |

**操作步骤**：

1. 编制资产包初稿，将资料表中信息按模板格式整理。
2. 标记信息缺失项，列出需要客户补充的材料清单，发送客户。
3. 客户补充材料后，完成资产包。
4. 发送客户审核。

### 阶段 5：GEO 内容生产

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 品牌事实资产包客户签字确认 |
| 负责人 | GEO 内容团队 |
| 输入 | 品牌事实资产包 + 诊断报告 |
| 输出 | [04_选题库](./04_GEO内容选题库模板.md) + 多篇内容稿件 |
| 周期 | 月度循环 |
| 标准版产能 | ≥4 篇/月 |
| 定制版产能 | ≥10 篇/月 |
| 客户动作 | 审核每篇内容稿件、给出修改意见 |

**操作步骤**：

1. 基于诊断报告的"信息缺口"和"优化建议"，规划首月选题（优先 P0）。
2. 与客户对齐选题后开始生产。
3. 每篇内容生产后发送客户审核。
4. 按客户反馈修改，定稿后进入发布阶段。
5. 每月更新选题库状态。

### 阶段 6：发布与分发建议

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 内容稿件客户审核通过 |
| 负责人 | GEO 交付 |
| 输出 | 内容上线 + 分发建议文档 |
| 客户动作 | 按建议发布内容（或授权 MOY GEO 代为发布） |

**分发建议维度**：

1. 客户自有平台：官网 FAQ、帮助中心、Blog、公众号
2. 第三方高权重平台：知乎、掘金、CSDN（根据行业和内容特性选择）
3. Schema 标记部署：确保页面包含 FAQPage / Article / HowTo 等结构化数据标记
4. 提交 URL 到搜索引擎和 AI 平台的内容收录入口

### 阶段 7：月度监测

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 进入月度服务周期 |
| 负责人 | GEO 交付 |
| 输出 | [05_月度监测报告](./05_月度监测报告模板.md) |
| 耗时 | 每月最后 5 个工作日 |
| 客户动作 | 接收报告 |

**操作步骤**：

1. 每月固定日期执行一次全平台 AI 可见度测试。
2. 对比上月数据，填写趋势和变化。
3. 监测竞品动态。
4. 检查是否有新出现的错误信息。
5. 输出月度报告。

### 阶段 8：月度复盘

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 月度监测报告完成 |
| 负责人 | GEO 顾问 |
| 输入 | 月度监测报告 |
| 输出 | 复盘会议纪要 + 下月计划 |
| 耗时 | 1 小时会议 |
| 客户动作 | 参加复盘会议、确认下月计划 |

**复盘会议议程**（30-45 分钟）：

1. 本月数据回顾（5 分钟）
2. 内容效果讨论（10 分钟）
3. 竞品变化讨论（5 分钟）
4. 下月优化方向对齐（10 分钟）
5. 客户新需求和反馈（10 分钟）

### 阶段 9：续费 / 升级

| 项目 | 内容 |
| --- | --- |
| 触发条件 | 合同到期前 30 天 |
| 负责人 | 销售 / GEO 顾问 |
| 输入 | 月度监测报告汇总 |
| 输出 | 续费合同 / 升级方案 |
| 客户动作 | 确认续费或升级 |

**续费沟通要点**：

1. 汇总服务期间的关键成果（数据+事实）。
2. 展示可见度趋势图。
3. 提出续费方案或升级建议。

## 4. 异常处理

| 场景 | 处理方式 |
| --- | --- |
| 客户迟迟不提供资料 | 3 个工作日提醒一次，7 个工作日升级沟通，15 个工作日暂停服务 |
| AI 平台回答出现严重偏差 | 24 小时内告知客户，48 小时内给出修正方案 |
| 竞品出现大规模内容攻势 | 立即预警客户，1 周内输出应对建议 |
| 客户对内容质量不满意 | 记录具体不满点，1 周内免费重做一次 |
| AI 平台大规模改版 | 重新执行全平台诊断，调整内容策略 |
