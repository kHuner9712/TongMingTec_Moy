## 一、文档目的

本文档定义 MOY 终局版系统中所有状态机的完整定义，确保：

1. 全量状态机统一管理、可追溯
2. 前后端状态定义一致
3. 状态变更可审计
4. 防止非法状态流转
5. 支持状态变更通知与自动化触发
6. 为终局版扩展提供规范化模板

---

## 二、适用范围

MOY 终局版全量业务，涵盖：

- 首期核心业务模块（线索、商机、客户、工单等）
- 终局版新增业务模块（合同、订单、回款、续费等）
- 扩展业务能力（转介绍、客户健康度等）

## 三、术语定义

| 术语 | 定义 |
|------|------|
| 状态机 | 定义实体生命周期中所有可能状态及其流转规则的模型 |
| 状态 | 实体在某一时刻的业务属性值，反映其业务阶段 |
| 流转 | 状态从一个值变更到另一个值的过程 |
| 触发事件 | 导致状态流转的业务操作或系统事件 |
| 终态 | 不可再变更的状态，表示实体生命周期的结束 |
| 权限要求 | 执行状态流转所需的用户角色或权限 |
| 业务规则 | 状态流转时必须满足的业务条件 |

#### 5.1.1 状态定义

| 状态值 | 中文名 | 说明 | 数据库值 |
|--------|--------|------|----------|
| new | 新线索 | 刚创建，未分配 | `new` |
| assigned | 已分配 | 已分配给销售人员 | `assigned` |
| following | 跟进中 | 销售正在跟进 | `following` |
| converted | 已转化 | 转化为商机或客户 | `converted` |

| assigned | following | 开始跟进 | assigned_user | - |
| assigned | invalid | 标记无效 | admin, sales_manager, assigned_user | 需填写无效原因 |

| 合同到期前30天 | 自动创建续费状态记录 |
| 客户状态为 active | 自动分配给客户成功经理 |
| 客户状态为 silent | 自动分配给销售 |

### 6.5 转介绍状态机

#### 6.5.1 状态定义

| 状态值 | 中文名 | 说明 | 业务含义 |
|--------|--------|------|----------|
| pending | 待联系 | 转介绍记录创建 | 等待联系被介绍人 |
| contacted | 已联系 | 已联系被介绍人 | 正在跟进 |
| converted | 已转化 | 转介绍成功 | 被介绍人成为客户 |

#### 6.5.2 状态流转矩阵

| 当前状态 | 目标状态 | 触发事件 | 权限要求 | 业务规则 |

| pending | invalid | 标记无效 | sales_rep, customer_success | 需填写无效原因 |
| contacted | converted | 转化成功 | sales_rep, customer_success | 需关联新客户/商机 |
| contacted | invalid | 标记无效 | sales_rep, customer_success | 需填写无效原因 |
| converted | - | 终态 | - | 不可变更 |
| invalid | - | 终态 | - | 不可变更 |

#### 6.5.3 奖励规则

| 状态变更 | 奖励动作 |
|----------|----------|
| pending → contacted | 记录跟进积分 |
| contacted → converted | 触发转介绍奖励（积分/优惠券等） |
