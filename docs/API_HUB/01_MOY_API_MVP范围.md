# MOY API MVP 范围

## 1. 文档用途

本文档精确定义 MOY API 第一阶段的开发范围，明确做与不做，作为开发排期和需求评审的依据。

## 2. MVP 目标

**用一个最小的技术闭环，验证"统一 API 网关 + 调用治理"的产品价值。**

MVP 交付后应能做到：

1. 开发者在 api.moy.com 注册登录
2. 创建一个 API Key
3. 用这个 Key 调用 `/v1/chat/completions`，选择模型
4. 在后台看到每次调用的 token 用量和成本
5. 额度用完后被限流

## 3. 做——MVP 必做清单

### 3.1 用户体系

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 邮箱注册 | 邮箱 + 密码注册 | P0 |
| 邮箱登录 | 邮箱 + 密码登录 | P0 |
| Session 管理 | JWT token，有效期管理 | P0 |
| 基础信息 | 显示名称（nickname） | P1 |

暂不做：手机号注册、OAuth 第三方登录、企业认证、邀请制。

### 3.2 API Key 管理

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 创建 API Key | 用户在前端创建，服务端生成，仅展示一次 | P0 |
| 列表查看 | 查看所有 Key，显示 prefix + 创建时间 + 状态 | P0 |
| 启用/禁用 | 一键切换 Key 状态 | P0 |
| 删除 Key | 软删除或硬删除 | P1 |
| 备注/标签 | 为 Key 添加说明 | P2 |

### 3.3 模型列表

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 模型清单 API | `GET /v1/models` 返回可用模型列表 | P0 |
| 模型信息 | 模型 ID、供应商、描述、上下文窗口、价格 | P0 |
| 模型可用性标记 | 标注模型当前状态（可用/维护中/已下线） | P1 |

### 3.4 Chat Completions 接口

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| `POST /v1/chat/completions` | OpenAI 兼容的非流式聊天补全 | P0 |
| model 参数 | 支持选择可用模型 | P0 |
| messages 参数 | 标准 `[{role, content}]` 格式 | P0 |
| temperature 参数 | 0-2 浮点数 | P0 |
| max_tokens 参数 | 限制输出 token | P1 |
| 上游错误透传 | 当供应商返回错误时，转为 OpenAI 兼容错误格式返回 | P0 |

### 3.5 调用日志

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 调用记录写入 | 每次 API 调用写入 usage_log 表 | P0 |
| 日志查询 | 后台按时间范围、模型、状态筛选 | P0 |
| 日志详情 | 查看单次调用的完整信息（不含 message content） | P1 |

### 3.6 用量统计

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 概览面板 | 当日/当月/总计：调用次数、token 消耗、成本 | P0 |
| 趋势图 | 按天的调用量/token/成本趋势（简单图表） | P1 |
| 模型维度 | 按模型分组统计 | P1 |

### 3.7 额度与余额

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 默认额度 | 新注册用户赠送固定额度（如 ¥10 等价 token） | P0 |
| 额度扣减 | 每次调用按实际成本扣减额度 | P0 |
| 额度查询 | 用户可查看当前剩余额度 | P0 |
| 额度不足提示 | 额度低于阈值时前端提示，额度耗尽时拒绝调用 | P0 |

### 3.8 基础限流

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 按 Key 限流 | 每个 Key QPS 上限（如 10 req/s） | P0 |
| 按 IP 限流 | 每个 IP QPS 上限（如 30 req/s） | P1 |
| 超限返回 | 返回 429 Too Many Requests + Retry-After header | P0 |

### 3.9 开发文档

| 功能 | 说明 | 优先级 |
| --- | --- | --- |
| 快速开始 | 从注册到第一次调用的完整指引 | P0 |
| API Reference | 接口参数、返回值、错误码说明 | P0 |
| 模型列表页 | 可用模型及定价 | P0 |
| 代码示例 | cURL / Python / Node.js 示例 | P1 |

## 4. 不做——MVP 明确不做

| 项目 | 说明 | 计划阶段 |
| --- | --- | --- |
| 复杂模型路由 | 按成本/质量/延迟自动选模型 | S3 |
| 自动 fallback | 主模型失败自动切换备用模型 | S3 |
| 响应缓存 | 对相同请求缓存结果 | S3 |
| 语义缓存 | 对语义相似的请求复用结果 | S4 |
| Stream 支持 | SSE 流式返回 | MVP+1（紧接 MVP 之后） |
| Function Calling | 工具调用透传 | S2 |
| 多模态 | 图片/视频/音频输入 | S3 |
| 团队权限 | 多成员、角色、权限 | S4 |
| 企业套餐 | 企业级合约、SLA、账单 | S4 |
| 完整 billing | 充值、支付、发票、对账 | S3 |
| 私有化部署 | 客户侧部署 | S4 |
| 统一账号 | MOY App / MOY GEO 共享账号 | S5 |
| WebSocket | 实时推送 | 暂无计划 |

## 5. MVP 项目结构（推荐）

```
api-hub/                        # 独立项目目录（与 frontend/backend 同级）
├── packages/
│   ├── server/                 # NestJS API 服务
│   │   ├── src/
│   │   │   ├── main.ts
│   │   │   ├── app.module.ts
│   │   │   ├── common/         # 公共：guards, filters, interceptors
│   │   │   ├── modules/
│   │   │   │   ├── auth/       # 认证模块
│   │   │   │   ├── user/       # 用户模块
│   │   │   │   ├── project/    # 项目/Key 模块
│   │   │   │   ├── model/      # 模型列表模块
│   │   │   │   ├── proxy/      # 代理转发模块（核心）
│   │   │   │   ├── usage/      # 日志与统计模块
│   │   │   │   ├── quota/      # 额度模块
│   │   │   │   └── rate-limit/ # 限流模块
│   │   │   └── config/         # 配置
│   │   └── package.json
│   └── web/                    # 前端开发者门户（已存在 sites/api）
│       └── ...现有代码
├── docker-compose.yml
└── README.md
```

## 6. 数据库

MVP 阶段推荐独立 PostgreSQL 数据库 `moy_api_hub`，与 MOY App 的 `moy_app` 数据库分离：

| 数据库 | 用途 |
| --- | --- |
| `moy_api_hub` | MOY API 业务数据 |
| `moy_app` | MOY App 业务数据（已有） |

远期 S5 阶段如需统一账号，通过 API 调用或数据库联邦实现，不混用数据库。

## 7. MVP 不依赖 MOY App

MVP 阶段 MOY API 是完全独立的系统：

- 独立的用户体系（邮箱注册/登录）
- 独立的数据库
- 独立的部署

不与 MOY App 的前端 `frontend/` 或后端 `backend/` 产生代码级耦合。

## 8. 验收标准

MVP 完成后，以下场景必须全部通过：

| # | 场景 | 验收方式 |
| --- | --- | --- |
| 1 | 新用户邮箱注册 → 登录 → 看到后台 | 手动测试 |
| 2 | 创建 API Key → 复制 Key → 用 cURL 调用 /v1/models | 手动测试 |
| 3 | 用 Key 调用 /v1/chat/completions → 收到正常回复 | 手动测试 |
| 4 | 后台看到调用记录（时间、模型、token、状态） | 手动测试 |
| 5 | 额度从默认值扣减 → 显示剩余额度 | 手动测试 |
| 6 | 用错误的 Key 调用 → 返回 401 | 自动化测试 |
| 7 | 额度耗尽 → 调用返回 402/429 | 自动化测试 |
| 8 | 高频调用 → 触发限流 → 返回 429 | 自动化测试 |
