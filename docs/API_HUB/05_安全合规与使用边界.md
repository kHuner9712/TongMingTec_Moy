# MOY API 安全合规与使用边界

## 1. 文档用途

本文档定义 MOY API 的安全基线、合规红线和使用边界。所有团队（产品、开发、运维、销售）必须遵守。

## 2. 安全基线

### 2.1 API Key 安全

| 规则 | 说明 |
| --- | --- |
| 不明文存储 | 数据库只存储 `key_hash`（SHA-256），不存储明文 Key |
| 创建时仅返回一次 | 完整 Key 仅在创建时返回，前端需提示用户立即保存 |
| 不可二次查询 | 没有任何接口可以查询已创建的 Key 明文 |
| 传输加密 | 全站 HTTPS，禁止 HTTP 明文传输 |
| 最小权限 | API Key 只能调用 API，不能用于登录后台 |

### 2.2 认证与鉴权

| 规则 | 说明 |
| --- | --- |
| JWT 管理后台认证 | 后台登录用 JWT token，有效期 24h |
| Bearer Token API 认证 | API 调用使用 `Authorization: Bearer sk-moy-xxx` |
| 密码强度 | 注册密码最低 8 位，含大小写字母和数字 |
| 密码哈希 | bcrypt + salt，cost factor ≥ 10 |
| 登录失败限制 | 同一 IP 连续 5 次失败锁定 15 分钟 |
| 登录日志 | 记录成功和失败的登录时间为审计日志 |

### 2.3 数据安全

| 规则 | 说明 |
| --- | --- |
| 数据库隔离 | MOY API 使用独立数据库 `moy_api_hub`，不与 MOY App 混用 |
| 连接加密 | 数据库连接 TLS 加密 |
| 备份 | 每日自动备份，保留 30 天 |
| 敏感字段加密 | 必要时对 PII 字段做应用层加密 |
| 日志脱敏 | 错误日志不包含 API Key、密码、用户 content |

### 2.4 基础设施安全

| 规则 | 说明 |
| --- | --- |
| HTTPS only | 全站 HTTPS，HSTS 启用 |
| WAF | 生产环境接入 Web 应用防火墙 |
| DDoS 防护 | 接入 CDN / 云厂商 DDoS 基础防护 |
| 依赖扫描 | CI 中集成 `npm audit` / `snyk`，阻断高危漏洞 |
| 容器安全 | 镜像扫描，非 root 运行 |

## 3. 合规红线

### 3.1 禁止用途（一票否决）

以下用途**绝对不允许**，一旦发现立即封禁 Key、冻结账户，且不退款：

| 类别 | 具体场景 |
| --- | --- |
| 违法内容 | 生成违反中国法律的内容 |
| 欺诈与诈骗 | 生成钓鱼邮件、虚假合同、冒充他人身份 |
| 垃圾内容 | 批量生成垃圾评论、广告帖、SEO 垃圾页面 |
| 攻击与滥用 | 生成恶意代码、漏洞利用 payload、攻击脚本 |
| 色情与暴力 | 生成色情、暴力、恐怖主义相关内容 |
| 侵权 | 未经授权使用他人版权内容训练或生成 |
| 虚假信息 | 批量生成虚假新闻、虚假评价、谣言 |
| 网络爬虫滥用 | 利用 API 批量生成内容用于爬虫/刷量/水军 |
| 竞品狙击 | 恶意生成针对特定企业或个人的诽谤内容 |
| 政治敏感 | 涉及政治敏感话题，或可能危害国家安全的内容 |

### 3.2 可接受使用政策（AUP）

所有开发者注册时需同意 AUP，明确：

1. 不得将 MOY API 用于上述禁止用途
2. MOY 有权监控调用模式，发现异常可暂停服务
3. MOY 有权配合执法机关提供必要信息
4. 开发者对其应用中的 AI 生成内容承担全部责任
5. 开发者应在面向终端用户的产品中标注 AI 生成内容的性质

### 3.3 内容安全

| 阶段 | 措施 |
| --- | --- |
| MVP | 基础关键词过滤（prompt 中检测明文敏感词），依赖上游供应商的内容安全能力 |
| S2 | 接入内容安全服务（如阿里云内容安全、网易易盾），对 prompt 和 completion 做安全检测 |
| S3 | 建立风险评分模型，对异常调用模式自动预警 |

### 3.4 审计日志

MVP 阶段审计日志最小集：

| 事件 | 记录内容 |
| --- | --- |
| 用户注册 | email, ip, timestamp |
| 用户登录 | email, ip, timestamp, success/fail |
| API Key 创建 | project_id, timestamp, ip |
| API Key 禁用/启用 | key_id, timestamp, user_id |
| API 调用首次超限 | project_id, timestamp |
| 封禁操作 | admin_id, target_id, reason, timestamp |

## 4. 使用边界

### 4.1 明确的边界

| 边界 | 说明 |
| --- | --- |
| 面向开发者，不面向 C 端 | MOY API 是 B2D（面向开发者）产品，调用方应当是开发者/企业，不是终端消费者 |
| 仅提供 API，不提供界面 | 不封装 chatbot UI、对话界面等 C 端产品形态 |
| 仅支持文本 | MVP 不支持图片、视频、音频等多模态输入 |
| 仅限非流式 | MVP 不支持 SSE streaming |
| 仅提供模型调用，不提供训练 | 不提供 fine-tuning、embedding 训练等能力 |

### 4.2 服务可用性

| 项目 | MVP 目标 |
| --- | --- |
| API 可用性 | 99.5%（允许计划内维护） |
| 状态页 | sites/api 有状态指示组件，手动更新 |
| 故障响应 | 工作日 4h 内响应 |
| SLA 承诺 | MVP 不做 SLA 承诺，标注"Beta" |

### 4.3 数据隐私

| 规则 | 说明 |
| --- | --- |
| 不用于训练 | MOY 不会使用开发者的 API 调用数据训练模型 |
| 不共享给第三方 | 开发者数据不分享给其他客户或第三方 |
| 数据存储位置 | 中国大陆 |
| 数据删除 | 开发者可请求删除其账户及关联数据 |

## 5. 企业客户独立部署

企业客户如有更高安全或合规要求，可申请独立部署方案（S4 阶段提供）：

- 客户自有云账号部署
- 独立数据库和密钥管理
- 可接入客户自有模型供应商账号
- 定制化内容安全策略

MVP 阶段不做，但架构设计时预留独立部署的可能性（配置化供应商账号、容器化部署）。

## 6. 风控机制（S2+）

以下风控能力在 S2 阶段建设：

| 能力 | 说明 |
| --- | --- |
| 异常调用检测 | 单 Key 调用量突增、异常时间调用、多地 IP 跳跃 |
| 自动熔断 | 异常 Key 自动禁用，需人工解封 |
| 用量预警 | 额度低于阈值邮件通知 |
| 内容安全审核 | prompt/completion 经内容安全服务检测 |
| IP 黑名单 | 恶意 IP 自动加入黑名单 |

## 8. 安全检查清单

每次发布前必须通过：

- [ ] HTTPS 已启用，无 HTTP 明文接口
- [ ] API Key 哈希存储已验证（数据库无明文 Key）
- [ ] JWT secret 已配置且强度足够
- [ ] 数据库密码不为默认值
- [ ] 依赖安全扫描通过（无 critical/high 漏洞）
- [ ] CORS 配置仅允许白名单域名
- [ ] 错误响应不泄露数据库结构、堆栈信息
- [ ] 敏感接口有 rate limit
- [ ] 生产环境 debug 模式已关闭
